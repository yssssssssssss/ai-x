# 访谈认识论（知识建构观）

> 来源：用户研究部·徐凯《重新认识访谈——那些"隐秘"的访谈知识》（内部培训稿，2021/7/26）；并参考收录文章《如何做好深度访谈》《做用户访谈不简单，小心被用户带跑偏》（人人都是产品经理）。

> 一句话：访谈得到的不是"被发掘出来"的现成事实，而是访谈者与受访者在会话中**共同建构**出来的知识；这套认识论决定了你如何看待访谈数据、进而如何设计提纲、执行访谈与分析资料。
> 落点：`models/`。判据——**拆开就讲不通、贯穿全程**才成立：把"知识从哪来"这一前设抽掉，提纲、追问、转录、分析、效度验证就都失去了统一的解释根基（贯穿确定主题→采集→分析→验证）。
> 边界：models 描述「被研究的东西本身长什么样 / 好不好」——这里描述的是"访谈知识本身是什么性质"；「研究怎么做对」（访谈执行规范）属 `standards/` 与 `methods/toolbox/collection/`；「怎么编码 / 出意义」属 `methods/toolbox/analysis/`。

## 核心概念

访谈是一种以**形成知识为目的**的会话，区别于日常的、艺术的会话（如媒体采访、心理咨询）。其核心争议在于：访谈所得的知识究竟是"本已存在、等待被取出"，还是"在对话中被生产出来"。

本模型采取**知识建构观**：访谈知识不仅仅是被发现 / 挖掘 / 给予的，更是通过问答主动构建的；它依赖访谈者与受访者的人际互动、依赖产生它的具体情境、并以语言与叙事为媒介。这一立场直接对应到访谈员的自我定位——是"矿工"还是"旅行者"（见下文"关键构念"）。

## 理论来源

访谈知识建立在三大哲学传统之上（源自徐凯培训稿"访谈的概念及知识"）：

- **现象学（Phenomenology）**：关注意识、经历、行为；主张从行动者的视角理解社会现象，**以被试所体验的方式描述其世界**。
- **诠释学（Hermeneutics）**：关注解释、含义、意义；即对行为意义的解释、对文本含义的解释。
- **建构主义（Constructivism）**：关注互动、情境；强调特定情境中的知识及意义，以及人与人之间的互动过程。

## 关键构念 / 维度

### 一、访谈知识的七个特征

徐凯培训稿将访谈所得知识概括为七个相互关联的特征：

| 特征 | 英文 | 含义 |
| --- | --- | --- |
| **产生的知识** | Produced | 不仅仅是被发现 / 挖掘 / 给予的，更是通过问答**主动构建**出来的 |
| **相关的知识** | Relational | 是受访者人际互动的知识，也是研究人员与受访者**观点互动**的知识 |
| **会话的知识** | Conversational | 通过会话对日常经验进行描述和叙事，并形成正确的知识 |
| **情境的知识** | Contextual | 人的生活和见解是情境性的，知识的形成和分析也**依赖于产生的情境** |
| **语言的知识** | Linguistic | 语言是访谈的媒介 / 工具，访谈的分析也是对语言和文本的分析 |
| **叙事的知识** | Narrative | 访谈注重发掘那些能够告知我们意义和知识的**具体的故事** |
| **实用的知识** | Pragmatic | 指向的是务实的、有用的知识 |

### 二、访谈员的两个比喻：矿工 vs 旅行者

源文用两个隐喻刻画访谈员的两种认识论立场：

| 比喻 | 对知识的假设 | 对应取向 |
| --- | --- | --- |
| **矿工（Miner）** | 知识存在于受访者内部，被理解为地下埋藏的金属，等待被发掘 | **知识采集**（Knowledge Collection） |
| **旅行者（Traveler）** | 知识本不存在，在旅行过程中通过一个个互动情境、故事而构建出来 | **知识建构**（Knowledge Construction） |

本模型整体倾向"旅行者 / 知识建构"一端，但两者并非全然对立——具体到一次访谈，研究者对"客观事实信息"与"意义建构"可能同时持有两种态度。

### 三、三大社会学视角

收录文章《如何做好深度访谈》给出了深度访谈的社会学理论基础——其共识是"一切现象都涉及人所赋予的意义"，并由此延伸出三种研究视角：

| 视角 | 聚焦 | 访谈中的着力点 |
| --- | --- | --- |
| **符号互动论** | 人的**主观意义** | 认知与理解来源于人与社会的互动；访谈中需重构研究对象用以解释世界的主观理论，把握其赋予行动、事件、经历的主观意义 |
| **常人方法论** | **缄默知识** | 即人们无意识的日常做事方式；访谈中需找出用户这种缄默知识，理解其无意识的日常行为模式 |
| **结构主义模式** | **文化 / 制度的影响** | 人的行为受文化、制度环境制约；访谈中需透过表层意识，发掘深层无意识文化模式的影响，研究用户态度的地域差异 |

### 四、人种学访谈的定位

收录文章《做用户访谈不简单》指出：用户访谈源于**人种学访谈**。人种学意味着系统深入地研究人类文化——人种学专家花费数年生活在所研究的文化中，去试图理解整个文化的行为和社会效益；**用户访谈借用其精髓，运用在微观层面**，目的是去理解与个体产品相关的人的交互行为和习惯。

> 由此可见，"理解情境中的人与文化"是访谈认识论的方法论根源；师徒式访谈（Master/Apprentice Model，人种学访谈 + 沉浸式观察的结合）即是这一取向在执行层的延伸。

## 如何作为透镜使用

这套认识论不是抽象哲学，而是会**改变你在三个阶段的具体动作**——核心是：既然知识是在互动中建构的，研究者就要为"建构出好的知识"负责。

- **问题定义 / 提纲设计阶段**：
  - 把"研究问题"与"访谈问题"分开——访谈问题要**简短易懂、能激励被试做自然的描述**（源文"问题评估"的主题维度 vs 动态维度）。
  - 由于知识是**叙事的、情境的**，提纲应引导受访者讲"具体的故事 / 还原场景"，而非索取抽象结论。落地为提纲骨架时可配合 ORID 等框架（见 `models/orid.md`）。
  - 由于知识是**会话的、相关的**，问题顺序要服务于互动节奏——"为什么"类问题后置，先取描述、再做判断（源文"问题顺序"）。
- **采集 / 访谈执行阶段**（详见 `methods/toolbox/collection/interviews.md`）：
  - 执行的三条主线是**围绕主题、保持互动、知识建构**——心中有研究全局、积极倾听保持动态自然，并以现象学 / 诠释学 / 建构主义为指引去"反思构建知识"。
  - 用分层的问题类型推进建构：导入性、跟踪性、探索性、说明性、解释性、结构性、间接性、直接性问题（源文"问题类型"）。
  - 因符号互动论 / 常人方法论的提示，要主动去**重构受访者的主观意义、捕捉其缄默知识**，警惕"用户见人说人话"的失真；因结构主义提示，要留意文化与地域差异。
- **分析 / 结论阶段**（详见 `methods/toolbox/analysis/qualitative-coding.md`）：
  - **永远不要在访谈完成后才考虑如何分析**——用分析方法反过来指导提纲准备、访谈过程与转录工作，把分析融入研究全过程。
  - 转录不是文案工作而是**一种解释过程**；录音已丧失姿态表情、转为文字又丧失语调节奏，故转录标准要"以使用目的来制定"。
  - 因知识是**建构的**，"真正的意义"是个伪命题——同一份资料因不同前验知识 / 分析层级 / 分析情境会有多种合理解释；研究者具有决定权，但**必须明确阐述所做解释的证据和理由**。
  - 落到具体技术：意义凝聚、意义编码、意义解释（概念驱动 vs 数据驱动编码），以及成员 / 观众 / 同行三类效度验证。

## 局限与误用

- **滑向"沉浸式聊天"**：把"知识在互动中建构"误读为"随便聊"。专业性不足时，访谈会因无结构而陷入无止境的话题衍生，记录到的大多是无效内容（《做用户访谈不简单》："因专业性不足导致访谈变成沉浸聊天""无法掌控现场谈话局面"）。建构观不豁免"围绕主题、保持主导性"。
- **把受访者的话当客观事实**：用户所说永远是其"主观"，且受邀者常为奖励而权衡、"见人说人话"。需借助追问、前后逻辑一致性检验等去伪（《做用户访谈不简单》"用户见人说人话"；尼尔森："基于自身经验提出的意见，可信赖度很低"）。
- **诱导与立场污染建构过程**：因知识是会话与相关的，访谈者的倾向性提问会直接"参与建构"出失真的知识——故须避免引导性、保持价值中立（《如何做好深度访谈》"访谈原则"；并见 `models/orid.md` 的诱导风险）。
- **以"意义是建构的"为借口逃避解释或放任随意**：源文强调解释的多元性是被允许的，但**前提是明确阐述证据与理由**；报告阶段常见"担心偏颇、逃避解释"与"盲目跟风、缺少质疑"两种失误，均应避免。
