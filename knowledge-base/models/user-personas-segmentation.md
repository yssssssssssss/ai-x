# 用户画像 / 用户分层 / 用户分群（三概念辨析）

> 一句话：精细化运营的三个紧密关联但各有侧重的概念——**画像回答"用户是谁"，分层回答"用户处于什么价值阶段"，分群回答"同阶段内的行为差异"**。
> 落点：`models/`。判据——这是"用户细分"工作的概念透镜，贯穿用研、运营、产品策略全程，三概念互相依赖、拆开就讲不通。

## 核心概念

### Persona vs Profile（必须先区分）

"用户画像"在实践中有两种本质不同的含义，混用会导致方法论错位：

| 维度 | Persona（定性用户角色） | Profile（定量用户画像） |
|---|---|---|
| **数据来源** | 定性访谈、用研 | 行为数据、埋点 |
| **产出形式** | 人物角色卡片 | 标签体系 / 画像数据库 |
| **使用场景** | 产品设计决策 | 精细化运营 |
| **更新频率** | 低（随产品阶段更新） | 高（实时 / 按天更新） |
| **用户数量** | 3 个以内典型角色 | 全量用户 |

**关键认知**：
- 用户画像 ≠ 用户细分。**用户细分**是基于销售数据的市场分类，偏重解释销售现状；**用户画像**关注"用户是什么样的人"，为产品设计服务。
- 设计阶段的决策应使用 **Persona（定性）**；运营阶段的圈人应使用 **Profile（定量）**。混用会导致：设计没有同理心，运营缺乏精准度。
- Persona 的具体构建方法（Cooper 法）见 [`methods/toolbox/analysis/persona-building.md`](../methods/toolbox/analysis/persona-building.md)。

### 三步递进

> **用户画像 → 用户分层 → 用户分群** 三步递进；每一步都建立在前一步基础上。

### 1. 用户画像

> 对用户**基础属性的全面刻画**——回答"用户是谁"。

通常包含 **六大维度**：

| 维度 | 说明 |
|---|---|
| **身份特征** | 年龄、性别、地域、职业、学历等人口统计属性 |
| **行为特征** | 访问频次、功能使用路径、停留时长等操作行为 |
| **消费特征** | 消费金额、消费频次、消费品类偏好 |
| **心理特征** | 动机、价值观、风险偏好、决策方式 |
| **兴趣爱好** | 内容偏好、娱乐偏好、社交习惯 |
| **渠道属性** | 来源渠道、设备类型、触达偏好 |

**用户画像是用户分层和分群的数据基础。**

### 2. 用户分层

> 基于用户在产品中**所处的状态**，将用户划分为不同**价值层级**，通常体现出明显的**递进关系**（如生命周期阶段）。

- 典型模型：**AARRR 生命周期模型**（拉新 → 激活 → 留存 → 收益 → 推荐）。
- 典型层级：新用户 → 活跃用户 → 流失用户。
- 核心目的：⚠️ 待补充(缺源)——原文标题为"核心目的："但具体内容缺失。

**用户分层三大模型**：

| 模型 | 适用 | 核心逻辑 |
|---|---|---|
| **AARRR 漏斗** | 增长分析 | 拉新 → 激活 → 留存 → 收益 → 推荐 |
| **RFM 价值分层** | 电商 / 交易类 | R 近度 + F 频度 + M 额度 → 8 类用户群体 |
| **用户生命周期分层** | 通用 | 按用户与产品关系深度划分 |

→ 详细 RFM 见 [`methods/toolbox/analysis/rfm.md`](../methods/toolbox/analysis/rfm.md)；AARRR 漏斗 / 生命周期分层待新建独立分析文件。

### 3. 用户分群

> 基于用户的**具体行为表现**对分层的进一步**精细化划分**，更聚焦于行为特征的**横向差异**。

- 典型模型：**RFM 模型**——见 [`methods/toolbox/analysis/rfm.md`](../methods/toolbox/analysis/rfm.md)。
- 典型分群：高频低额 vs 低频高额。

### 分层 vs 分群

| 维度 | 用户分层 | 用户分群 |
|---|---|---|
| **依据** | 用户价值状态（递进关系） | 用户行为特征（相互独立） |
| **典型模型** | AARRR 生命周期模型 | RFM 模型 |
| **目的** | 判断用户处于哪个价值阶段 | 找出同一阶段内的行为差异 |
| **案例** | 新用户 → 活跃用户 → 流失用户 | 高频低额 vs 低频高额 |

### 三者关系

```
用户画像（提供基础数据）
   ↓
用户分层（确定价值层级）
   ↓
用户分群（细化行为差异）
   ↓
共同支撑精细化运营策略制定
```

## 关键构念 / 维度

### 用户细分的五大维度

| 维度层 | 名称 | 性质 |
|---|---|---|
| **基础层** | 人口统计维度（年龄/性别/地域...） | 静态标签，粒度粗，需与其他维度结合 |
| **核心层** | 行为特征维度（产品中的真实轨迹） | 最真实的需求信号 |
| **深度层** | 心理特征维度（动机/价值观） | 难获取但洞察价值最高；通过问卷调研 + 行为推断 |
| **场景层** | 需求/场景维度（在什么场景使用产品） | 与产品设计结合最紧密 |
| **分层** | 价值贡献维度（按对产品/业务的贡献分层） | RFM 也属于这层 |

⚠️ 待补充(缺源)：每一维度下的具体子标签清单原文有但未完整捕获。

### 标签体系构建

- **三种基本类型**：实际业务中**统计类 + 规则类**标签通常能满足大部分运营需求；**算法类**标签主要用于预测性场景（推荐、风控）。
  - 🤖 草稿待审(AI生成)：常见三类是 统计类（如"近 30 天活跃天数"）/ 规则类（如"购物车有商品 + 7 天未下单 = 下单犹豫用户"）/ 算法类（如 ML 预测的"高流失风险"）——待 owner 核验。
- **树形多层级结构**：优秀的标签体系是树形多层级（一级 → 二级 → 三级 → 具体标签值）。
- **标签粒度原则**：粒度太粗 → 没有区分度；粒度过细 → 体系复杂、失去通用性。
- ⚠️ 待补充(缺源)：静态 vs 动态标签 / 显性 vs 隐性标签的具体内容原文缺失。

### 三维画像模型（递进认知框架）

| 层 | 名称 | 类型 | 内容 |
|---|---|---|---|
| 第一层 | **信息画像** | 静态 | 用户是谁——人口属性 |
| 第二层 | **行为画像** | 动态 | 用户做了什么——行为轨迹 |
| 第三层 | **分群画像** | 洞察 | 为什么这样做——动机 + 群体特征 |

**洞察链路**：信息画像告诉你"用户是谁"，行为画像告诉你"用户做了什么"，分群画像告诉你"为什么这样做"——**三层叠加才能找到真正的用户需求洞察点**。

### 用户分层体系搭建的两大原则

#### 1. MECE 原则：相互独立，完全穷尽

来自麦肯锡分析法则，要求分层结果：

- **相互独立（Mutually Exclusive）** —— 同一用户不能同时落在两层
- **完全穷尽（Collectively Exhaustive）** —— 所有用户都能落进某一层

**操作方法**：以产品目标为起点，逐层向下拆分行为节点，直到所有细分状态都被覆盖。

#### 2. 阶段目标原则：分层随产品生命周期动态调整

产品处于不同生命周期阶段，分层粒度和指标阈值都应相应调整。

- 早期：活跃阈值低，分层粒度粗。
- 功能完善后：相应提升标准、做更细分层。
- ⚠️ 待补充(缺源)：原文未给出各生命周期阶段对应的分层粒度建议。

## 如何作为透镜使用

### 问题定义阶段

- 把"用户怎么样"翻译成具体的**画像维度 / 分层指标 / 分群行为差异**——三选一或三结合。

### 采集 + 分析阶段

- 画像采集：问卷 / CRM / 行为日志。
- 分层：用 AARRR、RFM 等模型 + 行为漏斗。
- 分群：用 RFM、聚类等。

### 结论阶段

- 验证 MECE：所有用户是否都被覆盖？是否有重叠？
- 阈值是否需要随产品发展再调？

## 局限与误用（六大常见误区）

1. **把自己当用户** —— 产品经理最容易犯。Persona 的核心价值是"让你为一个具体的人设计，而不是为脑中虚构的东西设计"。每次讨论需求时应问：**我们的 Persona A 会遇到这个问题吗？**
2. **标签越多越好** —— 标签体系过于庞大会维护成本高、含义模糊、业务无法消化。应**以业务目标驱动**标签建设，明确每个标签的使用场景和消费方再建立。
3. **画像一劳永逸** —— 用户行为随时间、产品阶段、外部环境持续变化。**典型反例**：QQ 早期用户大量成家立业，不更新画像会导致产品决策严重偏差。**建议季度级审查、年度级重建。**
4. **只用人口属性细分** —— 仅用年龄、性别分群是最粗糙的方式，无法真正洞察需求，需叠加行为维度和心理动机。
5. **Persona 和 Profile 混用** —— 设计阶段用 Persona（定性）、运营阶段用 Profile（定量），混用导致设计没同理心、运营缺精准度（见"核心概念"）。
6. **画像与细分等同** —— 用户细分是手段，画像是目标。**过于依赖细分维度而忽视对用户目标和动机的理解，得到的只是数据分类，而不是真正的用户洞察。**
