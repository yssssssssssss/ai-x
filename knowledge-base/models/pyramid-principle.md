# 金字塔原理

> 一句话：用一座结论在顶、论据自下而上的"金字塔"组织思考与表达——结论先行、以上统下、归类分组、逻辑递进。
> 落点：`models/`。判据——它**贯穿研究全程**：界定问题用它（解决问题的逻辑）、思考整理用它（思考的逻辑）、汇报输出用它（表达的逻辑），拆开任一环就讲不通。
> 边界：models 描述"被研究对象本身长什么样 / 好不好"——金字塔原理的"被研究对象"是**用研者自己产出的报告 / 思考结构**，因此适合作为**研究输出物的设计原则**入 models（与"体验设计规范"作为产品输出物的设计原则同构）。

## 核心概念

- 一句话讲清：用"金字塔"组织信息——**中心思想明确，结论先行，以上统下，归类分组，逻辑递进**。先重要后次要，先全局后细节，先结论后原因，先结果后过程。
- 三个核心动作：
  - **自上而下表达** —— 写报告/汇报时从结论开始，层层下钻支持。
  - **自下而上思考** —— 整理材料时把同类事实归组，向上抽象出共性结论。
  - **疑问—回答驱动** —— 顶部结论必然激发读者疑问"为什么是这个结论"，每一层下钻就是在回答上一层激起的疑问。

## 理论来源

- 提出者 / 出处：Barbara Minto（麦肯锡），《金字塔原理：思考、表达和解决问题的逻辑》。
- 本条目正文整理自：[Joyspace - 金字塔原理...让工作汇报和表达更清晰](https://joyspace.jd.com/pages/4jS7PD87PmWiIW5DGz1h)（曾雨雯整理）。

## 关键构念 / 维度

### 1. 金字塔的纵向：疑问—回答

- 顶部一个中心思想 / 结论。
- 下一层每个论点都在回答上一层激起的"为什么"。
- 整篇文章 / 整次汇报 = 为了回答**序言中的那个疑问**而存在。
- **序言（SCQA）**：用 Situation（背景）/ Complication（冲突）/ Question（疑问）/ Answer（回答）四要素引出疑问。SCQA 四要素顺序不固定，关键是**激发读者疑问**——读者带着问题来读，才会被内容打动。

### 2. 金字塔的横向：归类分组 + 论证力度

每一层横向上的几个论据，必须满足：

- **MECE**（相互独立、完全穷尽）：归类分组要"不重不漏"。
- **演绎或归纳，二者不能并存**：
  - **演绎**：大前提 + 小前提 → 结论（三段论）。要求**两个前提都准确**，否则推理无效。
  - **归纳**：从多个有限前提抽象出共性结论。归纳的多个前提**只支持不保证**结论——归纳的结论不具备演绎的有效性。
  - 难点：准确定义"共性"。**很多商业洞察的价值就在这一点创造性——能发现别人发现不了的共性。**

### 3. 表达的四种逻辑顺序

⚠️ 待补充(缺源)：原文写到"容易被理解的逻辑顺序有以下四种"但未列举具体四种。Minto 原著中通常是 **时间顺序 / 结构顺序 / 重要性顺序 / 演绎顺序**——但这是基于对原著的回忆，**未经原文核验，不写入正文**，待 PR 由 owner 补正。

## 如何作为透镜使用

### 问题定义阶段（解决问题的逻辑）

- 把研究问题用金字塔结构界定清楚——顶部是核心研究问题，下面是分解的子问题。
- 有效准确的问题界定能直接提高后续分析效率。
- ⚠️ 待补充(缺源)：原文提到"我们可以使用以下框架来明确问题"，但未给出具体框架——留 PR 由 owner 补。

### 采集阶段

- 访谈 / 问卷设计时，每一道题对应金字塔上的一个论据，确保题目能向上聚合到研究问题。
- 避免"什么都问一点"——题目背后没有论证树支撑就是堆砌信息。

### 分析 / 结论阶段（思考的逻辑）

- 框架搭建阶段不要纠结细节内容；写完后回过头审视：逻辑是否清晰？是否漏了思路？
- **语句必须用明确的词汇**：不要"提高收入、改进流程、重配资源..."这种含糊措辞。
- 标准：**只看标题就能理解该部分讲什么**，正文只是论证。
- 不知如何明确时——**加数字、量化目标或结果**。

### 汇报阶段（表达的逻辑）

- 经验不丰富时**优先自上而下**搭建：先骨架，再填血肉。
- "序言"是最重要也最难写的部分——动笔前没想清场景和冲突，写到一半就会跑题，陷入混乱修订。
- 在每个标题里**提炼思想精华**，而不是只写"现状 / 问题 / 建议"这种空标题。

## 局限与误用

- **演绎和归纳不可混用** —— 同一层横向论据要么全部是演绎链条上的环节，要么全部是归纳的并列前提。混着用会破坏论证力度。
- **归纳的结论不能当真理用** —— 它只是"被支持"的，反例一出就破。汇报时要明确告知论证类型。
- **过度套用结构** —— 金字塔只是组织信息的工具；如果研究本身的发现稀薄，再漂亮的金字塔也撑不起来——别用结构掩盖内容空洞。
- **跑题陷阱** —— 如果序言的"情景—冲突—疑问"没想清楚就开始写，整个金字塔会偏离起点；写完发现不对，修订成本极高。
