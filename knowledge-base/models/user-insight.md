# 用户洞察（认知与误区）

> 来源：神灯圈子·李笑欣《认知篇：如何做好用户洞察？先纠正这三个认知误区》 原文 http://xingyun.jd.com/shendeng/article/detail/54811

> 一句话：用户洞察不是"调研"出来的，而是一套"逻辑推导分析 + 研究者感受力"的思维方式——用它来看清痛点/竞品/调研数据背后的真实需求与可行动策略。
> 落点：`models/`。判据——**拆开就讲不通、贯穿全程**才成立：洞察的认知贯穿问题定义→采集→结论，是看待"什么才算洞察"的透镜，而非某一步的算法。
> 边界：models 描述「被研究的东西本身长什么样 / 好不好」；「研究怎么做对」属 `standards/`；「怎么算/出图」属 `toolbox/analysis/`。

## 核心概念

一句话讲清：很多时候做不好用户洞察，不是方法技巧不行，而是在第一步——"理解什么是用户洞察"上就出了偏差。用户洞察不是一种可遇不可求的"玄学"感觉，而是一套有章可循的思维和工作方式。

要做好用户洞察，首先要在认知上完成三步转变（源文总结）：

1. **深度思考**：别把痛点、竞品和调研数据直接当答案，它们只是原材料，需要你深度思考和加工。
2. **向上攀登**：分清结果、结论与洞察，并通过逻辑推导和连续自我提问，向上攀登。
3. **善用感知**：不要让你的同理心和感受力在大数据时代被忽视，把自己当成最重要的研究工具。

这三步转变对应三个最常见的认知误区，构成本模型的三个支柱。

## 理论来源

- **提出者 / 出处**：本文为京东用户研究通道·李笑欣《认知篇：如何做好用户洞察？先纠正这三个认知误区》（2025-11-28）的方法认知总结。
- **援引理论**：第三支柱"把自己作为方法"援引人类学学者项飙《把自己作为方法》——强调从个人经验和切身体验出发，将自我视为观察世界和分析问题的起点和工具；在信息爆炸、观点纷杂的今天，要通过"自我"这个通道去连接更广阔的世界。
- **相关概念**："大数据 vs 厚数据"——大数据告诉我们用户怎么行动，厚数据告诉我们用户怎么思考。

## 关键构念 / 维度

本模型由三个认知支柱构成。

### 支柱一：三个认知误区——洞察 ≠ 调研，而是"逻辑推导分析"

洞察不是调研来的，而是推导分析来的。用户洞察应该像破案一样：用户调研、竞品分析获得的信息只是"破案"的线索，这些线索能推导出什么结论，取决于研究员能做出什么样的思考和提问。三类常见误区：

- **搜集痛点 ≠ 用户洞察**：辛辛苦苦收集一箩筐痛点，却依然不知道下一步做什么。面对痛点要先思考：哪些是机会点？哪些是当前需高优解决的？差异化优势应建立在解决哪个痛点上？而不是锚定一堆痛点就埋头解决。更重要的是——有时看起来解决了痛点，体验却没提升，因为**用户的需求并不单纯站在痛点之上，而是站在用户的场景和需求里**。痛点可能只是表象需求，只有洞察到深层需求才能得出真正的解决方案。
  > 经典案例：用户说"手动除草太累了，我没有除草机"，表面痛点是缺一台除草机；只看这一层，方案就只剩"一台更好更轻便更便宜的除草机"。但用户真正的深层需求是"让我的草坪长得整洁美观且不需要自己除草"——洞察到这一点，思路就打开了：人工仿真草坪、生长缓慢的草籽、园丁雇佣服务等等。这就是为什么用户痛点并不完全等同于用户需求。
- **竞品调研 ≠ 用户洞察**：竞品分析能带来启发，但做竞品分析时不能是"找不同"的心态——找到的竞品差异化信息只是类似"问卷原始数据""访谈原声"一样的数据，并不等于洞察。需要继续提问求证："他们的目标用户和我们的完全一样吗？""这个功能满足了什么场景？这个场景在我们的用户中存在吗？""他们为什么做这个？背后的业务逻辑是什么？我们能直接迁移吗？"经过这一系列自我提问和求证，获得的才是符合自身的洞察。
- **用户调研 ≠ 用户洞察**：用户洞察的本质是一种"思考和研究判断"，而常用的调研方法只是获取原始信息的手段和起点。调研数据本身不会说话，往往还需结合后台行为数据、行业报告、业务动作等信息，进行交叉分析和深度思考，才能形成"洞察"。整理完访谈资料和问卷数据，其实只是洞察的开始。

### 支柱二：结果 → 结论 → 洞察（逐层递进）

我们经常把"结果"直接当成"结论"，甚至误以为是"洞察"。三者是一个逐层深入的过程：

| 层级 | 定义 |
| --- | --- |
| **结果** | 你直接观察或测量到的东西 |
| **结论** | 你对"结果"进行分析后得出的原因判断 |
| **洞察** | 你能指导业务行动的建议策略 |

两段递进路径：

- **从结果到结论**：别只描述数据现象，尝试对结论做高度凝练。高度凝练不是形式主义，恰恰是获取洞察的关键——它能帮我们跳出琐碎的数据现象，放大思考的范畴，从更广阔的视角发现可能的洞察。
- **从结论到洞察**：得出高度凝练的结论后，洞察才刚刚开始，是思考的起点。此时要继续不断提问"如何呢、又能怎"，并结合竞品情况、业务动作、后台数据等其他数据类型分析，做到逻辑自洽——既能说服自己也能说服别人，才有可能把结论转化为"真洞察"。

> 送礼场景示例（一条数据走完三层）：
> - **结果**："68%的用户会在生日/纪念日送礼物，超过传统节日送礼（47%），日常随机/无特定日期的送礼场景也占到36%。" 需求方拿到这个结果并不知道有什么用、有什么启发。
> - **结论**（再凝练一层）："用户对生日/纪念日这类更个性化且更有针对性的送礼场景关注度在提升，呈现出日常化、即时化的趋势；相比节庆送礼的'延迟满足'，这种'即时满足 + 高频日常'的送礼方式也正在被用户需要。"
> - **洞察**（"如何呢 又能怎"）："送礼场景营销上，除了通过传统节庆日打'送礼心智'，建议也重点围绕'生日/纪念日''日常送心意'打心智，比如'不用等节日 天天都是送礼日'，扭转部分用户'只有节日可以送礼'的心智，传达送礼不是任务、也可以有'顺手表心意'的日常仪式感；此外顺应'送礼即时性'趋势，联动即时零售相关品类拓展送礼选择。"

所以输出调研报告时，一定要多向自己提问——"这仅仅是结果还是结论，可以转化成什么样的洞察"，逐层思考推进，而不是只交付一个"都是描述性结果"的报告。

### 支柱三：把自己作为方法——研究员的感受力是宝贵的洞察方法

在大数据时代，我们容易沉迷"数字"，却忽略了最重要的工具——我们自己。定性研究常被认为"主观、不够客观"，但这恰恰忽略了研究员本身就是分析问题的方法之一。"把自己作为方法"可能恰恰是获得洞察的关键。两个行动建议：

- **别丢掉你的感受力，"用蘑菇思维思考"**：访谈中用户的一个犹豫、一个不经意的小动作、一句随口抱怨，都可能比一个量化分数包含更多信息。研究员在调研过程中的主观感受和敏锐触觉，本身就是获得洞察的重要方法——这不是"不客观"，恰恰是理解"人"的关键。
  > "蘑菇思维"小故事：疯人院里有位无论晴雨都拿着雨伞蹲在角落的病人，怎么劝都没用，医生们纷纷猜测他可能在下雨天受过心理创伤。直到有位医生也拿了把雨伞蹲在他旁边，病人才开口："你也是蘑菇吗？""是呀，我也是一只蘑菇。"病人点点头继续做蘑菇。过一会儿医生站起来走动，病人问"你不是蘑菇吗，怎么可以走来走去？"医生答"蘑菇当然可以走来走去！"病人想了想也站起来走走。又过一会儿医生开始吃东西，病人又问，医生理直气壮答"蘑菇当然也可以吃东西。"病人觉得好像也对，于是也开始吃东西。几个星期后，这位病人就像正常人一样生活了——虽然他还是觉得自己是一只蘑菇。
- **培养看见数据背后那个"人"的能力**：做决策时越来越看重问卷数据、后台数据，认为数据更有说服力，用户洞察似乎变成了"数字游戏"，殊不知我们正在失去看见数据背后"人"的能力。用户洞察不是收集更多数据，而是培养看到数据背后"人"的能力。好的洞察从来不是冰冷的数据游戏：大数据告诉我们用户怎么行动，而厚数据会告诉你用户怎么思考。
  > 银发人群案例：在一次银发人群使用电商购物 APP 的调研中，把叔叔阿姨请到访谈室真实观察其下单行为。给一位阿姨一个非常简单的搜索任务，她点开搜索框、明明已有输入光标在闪烁，却一直不停点击搜索框。问她发生了什么，她说"这个搜索框里的文字怎么删不掉，我怎么输入我想输入的东西啊"——此时我们才恍然大悟：对我们已习惯的搜索框灰色暗文提示，对银发人群只是一种无用的阻碍。这种发现，看任何后台数据都不会发现。

## 如何作为透镜使用

把三个认知支柱映射进研究全程——它约束的是"什么才算洞察、洞察从哪里来"，而非某一步骤的算法。

**问题定义阶段**

- 不把需求方给的"调研痛点/对标竞品"直接当答案。面对痛点先问：哪些是机会点？哪些需高优解决？差异化优势建立在解决哪个痛点上？（支柱一）
- 把"解决痛点"的命题改写为"洞察痛点背后的用户场景与深层需求"，避免只交付一台"更好的除草机"。

**采集阶段**

- 调研只是获取线索的起点，访谈资料/问卷数据整理完只是洞察的开始；有意识地同时准备后台行为数据、行业报告、业务动作以备交叉分析。（支柱一）
- 保留并记录研究员的"主观感受"：访谈中的犹豫、小动作、随口抱怨；现场观察数据背后那个"人"，而不只采集量化分数。（支柱三）

**分析 / 结论阶段**

- 自检"我在哪一层"：当前产出是结果、结论还是洞察？（支柱二）
- 从结果到结论先做**高度凝练**；从结论到洞察持续追问"如何呢、又能怎"，并结合竞品/业务/后台数据做到逻辑自洽，能说服自己也能说服别人，再落为可行动的策略建议。（支柱二）

## 局限与误用

> ⚠️待补充(缺源)：源文未单列"局限与误用"清单。以下为源文已明确点出的反模式（不外推）：

- **把痛点/竞品/调研数据直接当答案**：锚定一堆痛点就埋头解决，或以"找不同"心态做竞品对标，得到的只是原始线索而非洞察。
- **停在描述性结果**：只描述数据现象、不做高度凝练，把"结果"误当"结论"乃至"洞察"，交付一份"都是描述性结果"的报告。
- **得出结论即收工**：忽略"结论之后洞察才刚开始"，不追问"如何呢、又能怎"，未结合其他数据做到逻辑自洽。
- **唯数据论、丢掉感受力**：在大数据时代沉迷数字，失去看见数据背后"人"的能力，把定性研究者的主观感受误判为"不客观"而舍弃。
