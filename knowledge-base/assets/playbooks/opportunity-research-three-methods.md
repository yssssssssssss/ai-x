# 机会点研究三法：亮点放大 / 友商镜鉴 / 空白挖掘

> 经验卡片（playbook）—— 抽象后的打法，遇到"业务方知道结论但要研究增量"的需求时启用。
> 落点：`assets/playbooks/`。
>
> 边界：本卡片是**研究思路的速查与决策**，不是方法本体——具体执行参考链接的方法/模型。

## 内容

### 痛点研究 vs 机会点研究：双轮驱动

**问题诊断**：业务方常问"用户为什么不来逛 / 不买 / 不复购"，研究员调研两周给出答案"商品力要强、价格要更有竞争力"——这些**业务方早就知道**。**调研的增量价值在哪里？**

**应对**：把研究分两轴看——

| 视角 | 干什么 |
|---|---|
| **诊断面 = 痛点研究** | 过去和现在；守住下限，让我们不犯大错 |
| **增长面 = 机会点研究** | 未来和可能；抬升上限，让我们做对事情 |

→ **不是二选一，而是双轮驱动。**

### 三大挖掘机会点的方法

#### 1. 亮点放大法（Look Inside）

> **向内看做得好的地方，把它放大。**

找到内部正面案例，作为可进一步放大的增长路径。

**典型流程**：
1. 痛点研究找到：某类用户/商家做得不好。
2. 从这部分人里找**做得好的少数派**——他们是怎么做的？
3. 抽象出可被复制 / 推广的模式，赋能长尾。

**案例（商家店铺装修）**：
- 痛点视角：商家因为流量少 / 店内运营空间小，不愿在装修上投入。
- 机会视角：研究**已经在装修**的商家，提取**腰头部商家的装修逻辑**。
- 发现 4 类装修模板：

| 模板 | 适合 | 装修逻辑 |
|---|---|---|
| **爆品转化型** | 低频标品（主机电脑） | 以爆品为绝对核心；轮播图焦点大图展示卖点和价格；强化单品活动；最短转化路径 |
| **品类逛购型** | 多 SKU 但低频品类（眼镜、家居） | 清晰分类导航 + 全店连带活动 |
| **上新复购型** | 高频上新 / 强复购品类（服饰、首饰） | 顶部上新单品或系列；场景化导购强化逛感；会员权益锁复购 |
| **品牌形象型** | 成熟品牌 / 设计师品牌 / 奢侈品 | 弱化促销感；强化品牌视觉叙事（楼层视觉统一）；强调品牌特点而非货品堆叠 |

→ 通过商家特点（SKU 宽度 / 上新频率 / 品牌规模 / 营销配置），或对话式交互了解经营目标，**给长尾商家推荐符合其业务特点的装修模板**。

#### 2. 友商镜鉴法（Look Across）

> **向外看做得好的地方，学过来。**

研究友商做对了什么，给我们灵感。

**典型流程**：
1. 自家痛点研究确认问题是行业通病还是自家独有。
2. 看友商的解法——精细化运营？规模优势？算法能力？
3. 拆出可被本平台复制 / 借鉴的能力建设方向。

**案例（骑手体验）**：
- 痛点视角：骑手抱怨赚钱少、派单不合理、商家卡餐、时效压力大。
- 机会视角：访谈美团 / 饿了么骑手 → 发现这些是**行业通病**，但美团靠规模做了**精细化运营**：
  - 普众 / 畅跑 / 乐跑 / 乐跑远 / 专送几条运力线，在订单池和工作范围上互相补充
  - 困难单精细化定义和激励
  - 算法 ETA 准确度和动态加时
- → 自家的能力建设方向：长期精细化策略与运营（困难单激励 / 商家卡餐补时 / 报备与惩罚申诉机制）+ 提升 ETA 预估、派单算法、地图能力。

#### 3. 空白挖掘法（Find Gaps）

> **看友商没做好的地方，补上去。**

友商的痛点 = 我们的机会点。

**典型流程**：
1. 痛点研究确认自家差距（如价格力 / 丰富度）。
2. 同时**研究用户在友商那的痛点**——大概率有友商也没解决好的通用痛点。
3. 把自家已有优势对准这些通用痛点，建立差异化心智。

**案例（女装心智）**：
- 痛点视角：用户不来京东买女装的核心原因是价格力不够、丰富度不够（长期建设）。
- 机会视角：用户在淘天 / 抖音 / 拼多多购物都有通用痛点——**退货率过高，买买退退疲惫不堪**。
- → **京东的省心保障**正是解决其他平台痛点的切入点。差异化心智："**省下挑选和退货的精力，收获品质与从容**"。

## 使用说明

### 何时用

- 业务方提的问题是"用户为什么不来 / 不买 / 不复购"这类**已经知道大致答案**的问题——纯痛点研究价值不足时。
- 启动需求洞察类项目，**既要诊断也要找增长**。
- 与 [`methods/standards/research-question-definition.md`](../../methods/standards/research-question-definition.md) 配合使用——业务问题定义后，机会点研究是给增长面的子问题。

### 怎么用

按"痛点 → 机会"双视角组织研究：

1. 先用痛点视角搞清现状（避免为做机会而做机会）。
2. 同一份数据/同一批受访者，**叠加机会点视角**：
   - 内部正面案例（亮点放大）
   - 友商优势（友商镜鉴）
   - 友商通用痛点（空白挖掘）
3. 把"自家可放大"的、"友商已做对"的、"友商共同槽点"的三类机会汇总，结合业务能力做选择。

### 注意事项

> 三种方法的共同局限——**研究员不能造汽车，只能告诉造车的人用户想要什么样的车。**

#### 局限一：幸存者偏差（针对亮点放大法）

亮点用户**不能代表全部市场**。少数做得好的商家/用户，可能依赖了无法被复制的因素（资源 / 品牌力 / 运气）。把他们的方法推给长尾——可能水土不服。

#### 局限二：跟随者的困境（针对友商镜鉴法）

友商先做的功能 / 体验 → 它**定义了用户预期和心智**。你跟随做得再好，也只是"另一个也能做这个的平台"。**先发优势很难追赶。**

#### 局限三：填补空白 ≠ 赢得市场（针对空白挖掘法）

友商的短板可能是**用户愿意容忍的代价**——因为它的核心优势太强（拼多多太便宜，用户愿意容忍物流更慢、包装更差）。补短板可能用户根本不在乎，或者在乎但不足以让他离开友商。

### 研究员的边界

> **商业世界是复杂且混沌的，而研究是理性的。** 用理性工具理解或创造混沌世界，一定有局限。

研究员能做的：
- 把用户 / 竞品 / 数据的洞察带到桌上。
- 与合作方（产品 / 业务 / 设计 / 技术）一起碰撞——用研的视角 + 合作方的市场/业务/产品/技术理解。
