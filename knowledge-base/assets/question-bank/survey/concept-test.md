# 问卷题库 · 概念与方案评估（concept-test）

> **主题定义**：对具体方案/专区/功能/呈现的偏好与量化评价。收"对这个方案怎么看、更倾向哪个"。
> **占位符约定**：`[目标品类]`/`[目标人群]`/`[本平台]`/`[候选标签]`/`[候选规则]`/`[目标权益]`/`[目标场景/结果页]`/`[AI功能]`，复用时替换。

## 题组

### 专区组织方式偏好

- **【多选·限N项】假如 `[本平台]` 提供 `[目标品类/目标人群]` 专区，您觉得按什么方式分类用起来方便？** — 分类维度池（按年龄 / 按用餐情景 / 按功能 / 按品类类型 / 按口味 / 按做法 / 按菜系），限选 3 项以逼出优先级。
- source：qb3 Q16 / Q19 / Q22（三场景折叠）

### 期望服务

- **【多选·限N项】给 `[目标人群]` 用 `[目标品类]` 时，您最希望平台提供下列哪些服务？** — 服务设想池，限选 5 项。
- source：qb3 Q23
- **变体（理想方案开放畅想）**：请描述您理想中的 `[目标方案/AI功能]` 应具备哪些能力、有哪些特点、能帮您完成哪些具体的事？ — 开放题，非必答，作方案段收尾。
  - source：s5 AI动线 Q28 · s4外卖月卡 Q24 · s1商旅 Q25（开放建议标签/信息）

### 功能型 / 属性偏好

- **【多选】如果推出更 `[某属性]`（如"科学营养"）的 `[目标品类]`，您希望重点满足哪些目标？** — 目标池（如护眼 / 助消化 / 长高 / 益智 / 开胃 / 增强体质 / 专注力 / 控重…，"均不愿尝试"排他沉底）。
- source：qb3 Q15

### 拉新 / 转化举措偏好

- **【多选】下列哪些举措能吸引您来 `[本平台]` 购买 `[目标品类]`？** — 举措池（新客立减 / 首单包赔 / 小规格试吃 / 搭配满减 / 赠样…）。
- **【单选】以下哪种专区对您更有吸引力？** — 如 新品首发 / 地方特色 / 时令限定 / 销量排行榜 / 配料干净。
- source：qb6 渠道选择（吸引举措）+ 品类购买习惯（专区吸引力）
- **变体（竞品权益借鉴·开放）**：其他平台的月券包/月会员使用经历中，有哪些让您印象深刻、体验好的权益或玩法？哪些吸引到了您？ — 开放题，非必答。
  - source：s4外卖月卡 Q23

### 品质感 / 满意度打分（量表·需附图）

- **【描述题/量表·需附图】下面展示的分别是 `[本平台]` 与 `[竞品A]`/`[竞品B]` 的 `[页面]` 样式，请根据视觉感受，对以下各描述的符合程度 1–7 打分。** — 多平台 × 多描述的品质感矩阵评分。
- 说明：量表口径见 [[standardized-ux-scales]]；配图在问卷投放时插入。
- source：qb5 PART5（首页品质感感知）
- **变体（标签适配度视觉打分·需附图）**：请浏览以下店铺页，为各店铺与"`[候选标签]`"的适配度打 1–5 分。 — 店铺样例 × 适配度矩阵（配图投放时插入）。
  - 说明：与本组品质感打分同为"附图视觉评分"形态，对象是标签-店铺适配度；亦见 [[survey/mindset-cognition]] 的标签语义区分（同批 s2，交叉指引）。
  - source：s2店铺 Q22/Q26

### 新概念接受度测试

- **【单选】关于 `[某属性]`（如奶粉生产日期/新鲜度），下列哪个描述更符合您的想法？** — 态度分档（不在意→非常在意，用于分型）。
- **【单选】您是否听说过 `[新概念]`（如"28天新鲜"）或类似概念？** — 听说过/没听说过（认知漏斗）。
- **【单选】现在请了解一个概念：`[新概念定义]`。基于描述，您对它感兴趣吗？** — 非常感兴趣→不感兴趣（兴趣测试，附概念解释）。
- **【多选】您对 `[新概念]` 感兴趣/不感兴趣的原因是？** — 双向原因池（感兴趣：口感更好/营养保留/风险低/尝鲜/满足感/囤货无忧；不感兴趣：营销噱头/不清楚好处/无需极致/不信承诺/囤货用不上/担心涨价）。
- **【多选】以下哪些改进会让您愿意尝试 `[新概念]`？** — 改进举措池（权威检测报告/溯源直播/线下盲测/价格一致/订阅配送模式/博主实证）。
- source：奶粉 Q36/Q37/Q38/Q39a/Q39b/Q40（28天新鲜概念测试全链路）

### 新品 / 新品类尝试意愿 + 定价测试

- **【单选】您是否愿意尝试 `[新品类]`（如低度白酒/光瓶白酒）？** — 非常愿意→非常不愿意（5档意愿）。
- **【单选】如果 `[某品牌]` 推出 `[某规格某价位]` 的 `[新品]`，您是否愿意尝试？** — 5档意愿（具体产品概念测试）。
- **【多选】您不愿意尝试 `[新品类]` 的原因是？** — 阻碍池：健康顾虑/口感顾虑/场景不适配/品质偏见/价格不符/习惯（可复用的"新品类抗拒"归因）。
- **【多选】若尝试 `[新品类]`，您更偏好哪种 `[风味/口感/香型/包装风格]`？** — 各维度偏好池（风味：果味/茶味/草本/复合；口感：醇厚/绵柔/清新等；香型：酱香/浓香/清香等；包装风格：老字号经典/极简高级/小清新自然/国潮插画/潮玩酷炫/环保轻量）。
- source：光瓶酒 A14/A14b-d/A15/A16/A17/A18/A19 · 低度酒 A10/A11/A14/A15/A16/A17
- **【单选】您认为 `[新品]` 的定价应比 `[对标品]`？** — 低很多(8折以下)/略低(9折)/持平/更高（相对定价测试）。
- **【单选/多选】您比较倾向哪些价位段？** / **您认为 `[某规格某价]` 属于什么水平？** — 价位段池 + 贵贱感知（非常便宜→非常贵，PSM式）。
- source：低度酒 A18/A19 · 光瓶酒 A(价格水平感知)

### 品牌形象拟人化投射

- **【单选】如果把 `[目标品类]`（如光瓶白酒/盒装白酒）比作人，您觉得哪种形象最符合？** — 形象池（务实老友/街坊师傅/精打细算家人/不拘小节年轻人 // 商务人士/家族长辈/爱排场朋友/收藏家）。
- **【主观题】请用2-3个词形容一下 `[目标品类]`。** — 开放联想（非必答，附示例）。
- source：光瓶酒 A11/A12/A13

### 商品卡信息 / 价格展示测试

- **【多选·限N项·选项随机】浏览 `[目标品类/频道]` 商品时，您最关心商品卡片上的什么信息？** — 信息池：价格够不够便宜/是否包邮/销量/评价/是否品牌/是否自营/售后服务。
- source：特价 Q13
- **【多选·限N项·选项随机】看到哪种价格描述最让您觉得"划算、想点进去"？** — 价格话术池：原价锚点(原价¥X)/立减¥X/用券减¥X/限时5折/已补贴¥X/近30天最低价/比平时便宜X%/价同618双11/全网低价。
- source：特价 Q14
- **【多选·限N项·选项随机】看到哪种数据信息最让您觉得"靠谱、想点进去"？** — 数据标签池：热销榜第X/回头客X人/销量飙升X%/加购飙升/热搜度/已售X件/近3天卖X件/N分钟前有人买/仅剩X件/好评率。
- source：特价 Q16
- **【多选·限N项】您希望哪些服务标签优先展示在商品卡上？** — 服务承诺池：包邮/7天价保/买贵双倍赔/先享后付/免费上门退换/品类专属保障(生鲜坏损包退等)。
- source：特价 Q15
- **【单选】以下几张商品主图，哪张最有吸引力、最想点进去？** — 图片风格测试（白底图/场景图/牛皮癣/组合图/开袋图/细节图），需附图投放。
- source：特价 Q17-Q23（多品类主图测试，需附图）

### 折扣呈现 / 活动玩法偏好

- **【单选/多选】以下折扣形式，您更倾向哪种？** — 折扣形式池：直接降价/1元抢购/满减券/折扣券/买一送一/N元N件/全场包邮/满减阶梯/买N送1/赠品。
- source：特价 Q12 · 厨电 Q18 · 奶粉(赠品Q27)
- **【多选·限N项】哪些活动/任务最能吸引您每天来参与？** — 玩法池：每日签到/连续签到领大奖/每日抽奖/做简单任务/邀请助力/养成兑换/助力提现。
- source：特价 Q24/Q25 · 文具(活动玩法认知 B20)
- **【单选】`[某任务]` 给多少 `[京豆/红包]` 比较合适？** — 奖励额度测试（含当前锚点说明）。
- source：特价 Q27/Q28（奖励定价）
- **【多选·限N项】如果参加互动活动，哪些奖励最吸引您？** — 奖励池：京豆/无门槛红包/通用券/指定品类券/低价换购/实物奖品/现金红包。
- source：特价 Q26

### 会员新权益概念 / 联名卡测试

- **【填空】您希望 `[会员]` 增加哪些 `[某类]`（生活/餐饮/金融/出行/娱乐）权益，会更愿意续费？** — 分类开放收集（新权益脑暴）。
- source：PLUS B1/B2/B3/B5/B6 · C9/C11/C12
- **【多选】您希望 `[会员]` 增加哪些类型的联合会员卡？** — 联名平台池（音乐/视频/工具/运动健康/社交/读书/AI/餐饮/办公/打车/知识/教育/游戏/会议）。
- source：PLUS C10
- **【多选】您愿意开通 `[某联名卡/大屏卡]`（附定价与权益说明）吗？** + **【多选】愿意/不愿意的原因？** — 套餐概念测试 + 双向原因（联合卡优势/大屏优势 // 价格/需求不匹配/已有替代/设备限制/偏好灵活）。
- source：PLUS D1/D2/D3/D4/D5/D6
- **【单选】`[某权益]` 对您是否续费 `[会员]` 的影响是？** — 完全不重要→至关重要（权益对续费的重要性）。
- source：PLUS A6

### 套装 / 专区 / 组合方案偏好

- **【单选/排序】您认为一套 `[目标品类]` 套装应包含哪些？按优先级排序** + **【单选】合理的套装折扣应是多少？** + **【多选】您会因哪些原因放弃购买套装？** — 套装方案测试（组成/折扣/放弃原因）。
- source：厨电 Q13/Q15/Q16/Q17
- **【多选·限N项】您最希望 `[频道/平台]` 多提供什么样的商品？** — 货品期待池（一站式日消/产地直发/品类深度折扣/新奇小物/大牌低价/平价替代/临期清仓/极致低价）。
- source：特价 Q5 · 驱虫(AI推荐维度衔接)

### 候选标签 / 信息概念验证（态度四档 battery·可复用模式）

- **【单选·每个候选标签一题】在 `[目标场景/结果页]`，如果对符合要求的 `[目标品类]` 增加【`[候选标签]`】这个标签/信息，您的态度是？** — 固定四档：非常有用，应优先展示 / 有点用，但不用太突出 / 无所谓·没必要，不关注 / 我关注该方面，但单纯文字标签很难帮助判断。
- 候选标签池（示例·差旅酒店）：新开业/新装修、安睡乳胶/记忆枕、床垫软硬标签、加热马桶盖、早餐可外带、免费夜宵。
- 说明：对每个拟上线信息点/标签用统一四档量表测"是否值得展示"，`[候选标签]` 替换即可复用到任意标签/信息价值验证；第四档"关注但单纯文字标签很难帮助判断"是关键洞察选项，区分"不关注该维度"与"关注但不信任该呈现形式"。s1 对 6 个候选标签各问一遍（Q19–Q24，选项完全一致），此处折叠为 1 个题组模式。
- source：s1商旅 Q19–Q24（6 候选标签态度四档 battery 折叠）

### 机制 / 规则概念验证

- **【单选】如果 `[本平台]` 推出"`[候选规则]`"（如"支持出差补贴支付超额部分"），您的态度是？** — 态度分档（很方便，愿意用 / 中立，视情况 / 不喜欢，应单独结算）。
- 说明：与"标签概念验证"同为"拟推出方案的接受度"测量，但对象是规则/机制而非展示信息；可与前置行为题（如"遇到差标不够时怎么做"）配对做条件分析。
- source：s1商旅 Q26（前置场景行为）+ Q27（规则态度验证）

### 权益组合偏好（会员 / 权益类专用）

- **【多选·可设下限】如果 `[目标权益]`（如外卖超级月卡）升级，您更希望保留或增加哪些权益？** — 权益池（折扣券/急送券/免单抽奖/专属价/连续下单红包/超时赔付/生日券包/优先配送/其他服务券；选项随机，"其他"兜底）。
- **【多选·限选】如果在 `[目标权益]` 中附赠其他服务的优惠/权益，您对下列哪些更感兴趣？** — 跨品类服务池（家政/商品试用装/机票火车票/鲜花/免费菜/酒店升房/话费/洗衣/在线问诊；"以上都不感兴趣"排他）。
- source：s4外卖月卡 Q20/Q21

### AI 能力需求（"希望 AI 介入的场景"·与痛点配对）

- **【多选】在电商 App 逛购时，以下哪些情况您希望得到 AI 的帮助？（无需考虑当前 AI 能力）** — 场景池与"逛购麻烦点"题同一选项池配对（推荐不感兴趣/搜索/筛选/实际效果/对比/参数/评价/低价/凑单/客服/物流/售后；"没有希望AI帮助的"沉底）。
- 说明："现状麻烦（[[experience-painpoints]] Q9）→ 希望 AI 帮（本题）"的配对结构，同池对照可算出"痛点强度 vs AI 需求度"缺口。
- source：s5 AI动线 Q10（与 Q9 痛点池配对）

### 标签适配方案（哪类店铺 / 对象适合某标签）

- **【多选】您认为具有以下哪些特征的 `[目标品类]` 店铺更适合打上"`[候选标签]`"？** — 店铺特征池（大众知名品牌/红人主理店/风格流行小众店/特定文化风格店/买手集合店/大牌平替店；"其他"沉底）。
- **【单选】"`[候选标签]`"标签最应该出现在哪类店铺中？** — 自营/旗舰店 / 也可第三方 / 无所谓，看商品风格。
- source：s2店铺 Q21/Q23（潮流好货标签适配）

## 使用说明
- 怎么用：`generate-survey` 组装"方案/概念评估"段时取本文件；专区/服务/举措清单按项目替换，限选项数以产出优先级。
- 注意：概念题避免诱导（不预设"这个方案很好吧"）；量表题给清晰锚点与指导语。
- 自查：过 [[questionnaire-design]]（题干中性、锚点清晰、限选项数说明）。
