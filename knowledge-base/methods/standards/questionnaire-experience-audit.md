# 问卷体验审核规范

> 来源：Joyspace「问卷推送及体验审核规范」**第二部分「二、问卷体验审核规范」**（原稿留底 `joyspace-raw/f4-questionnaire-experience-audit.pdf`；第一部分「消息盒子推送规范」属推送运营流程、不在本库范围，未转录）。
>
> 内部「必须遵守」的权威规则。本规范在本库**单一出处**。
> 落点：`methods/standards/`。边界：`standards/questionnaire-design.md` 讲「问卷题目/选项怎么设计才对」（设计规则）；本规范讲「问卷对外推送前，**用户填答体验**必须过哪些审核」（发布前的体验门禁，含系统、欢迎/结束语、招募问卷、奖品、投放文案等设计之外的环节）。

## 适用范围

- 适用于用户研究部对外发出的**全部问卷**（包括精准推送与非精准推送）。
- **所有问卷内容、投放物料，均需经用户研究部内部审核通过后方可对外推送。**
- 目的：保证用研部发出的问卷在前台的**用户填答体验**。

## 必须遵守（MUST）

> 以下为发布前可硬性核对项。

1. **问卷系统**：问卷录入**必须使用京东自主研发的问卷系统**（洞见问卷系统、京东调研、京东良研），以保证数据安全；不得使用外部问卷系统。
2. **欢迎语必备且达标**：问卷**必须有欢迎语**，且必须包含——调研**主题**、**数据隐私保护条款**、**奖励信息**；欢迎语中承诺的信息（如预计花费 X 分钟、奖励内容）**必须与实际相符**；奖励信息**无歧义**，利益点准确清晰（含奖项设置、奖项内容及金额、可获奖人数、发放方式、发放时间等）。
3. **正文基本质量**：问卷正文**不允许出现错别字**；题干描述的**题型必须与实际设置一致**（严禁题干标「多选题」实际却是单选）；限选题、矩阵题等**必须标注清楚具体条件**（如「限选 3–5 个」「每行单选」）；题目之间的**跳转逻辑必须正确**。
4. **隐私题非必答 + 注明保护**：涉及用户隐私搜集的题型，**必须设置为非必答**，并在题干注明数据隐私保护话术（如「您的个人信息将仅用于本次用户招募，我们会保护您的个人隐私，避免信息泄露」）；开放题如非特殊情况须填答，也应设为非必答。
5. **结束语必备**：问卷结束**必须包含结束语**（如「非常感谢您的参与」），并再次说明奖励发放方式或明确引导用户参与抽奖。
6. **奖品即时发放**：推送问卷**一律设置即时发奖**（一般为抽奖形式），用户填答后即可获得奖励（京豆或实物）；奖品内容及发奖比例由发起方自行设置。
7. **中奖公示脱敏**：中奖信息如需公示，**必须对用户敏感信息（如 pin、昵称、联系方式、地址等）做隐藏保护处理**。
8. **招募问卷防误导**：「访谈招募」类问卷的投放标题与文案，**必须明确注明**「访谈活动报名」「面对面访谈」「电话访谈」等字眼，避免用户误以为「填问卷 = 接受访谈」。
9. **投放文案字数**：短信 67 个字符为 1 条、超过按 2 条计费；**APP Push 文案不超过 45 个字**。

## 推荐（SHOULD）

- **欢迎语**：字数不宜过多、符合用户阅读习惯；重点信息可标红突出。
- **正文可读性**：字号建议 **12–14 号**；题干及选项用语符合普通用户认知，不含生僻/专业/晦涩用语；交互方式方便点选——尤其排序题/矩阵题，需保证**移动端**填答体验。
- **甄别题控制**：甄别题建议控制在 **1–3 个**；如甄别题较多，建议**不设置提前结束**，让不符合条件的用户也能正常结束、参与抽奖。
- **招募问卷专项**（一般不设激励、极易引发客诉，需特别注意）：
  - 减少题量（甄别题 1–3 个；除报名信息外的题建议控制在 **10 题左右**）；
  - 降低填答难度，尽量少用态度类、量表/矩阵等高难度题型；
  - 不符合条件的用户也最好让其正常提交；如需甄别，甄别文案注意**亲和力**，可告知用户被甄别的原因；
  - 原则上招募问卷**不搭车「定量调研」**；如确需搭车，建议申请预算、以「调研问卷」形式推送并设置抽奖。
- **提前结束文案**：甄别「提前结束」的文案注意语言亲和力，可告知被甄别原因（示例：「不好意思，本次调研仅针对在京东购买过宠物用品的用户，您暂时不符合调研要求，再次感谢您的参与，祝您生活愉快」）。
- **投放文案结构**：
  - **APP Push 构成**：文案标题 + 活动背景简介 + 活动利益点 + 入口（落地页为问卷链接）；
  - **短信构成**：短信签名（系统自带）+ 活动背景简介 + 活动利益点 + 活动截止日期 + 落地页链接（系统生成短链）；
  - 文案要点：活动类型清晰（如电话访谈/面对面访谈报名）、利益点无歧义（写明京豆/数额/京东卡等，措辞准确）、标题简洁、不出现专业生僻或复杂缩略词/英文（如「品类」「NPS 测试」「Q1」等不易理解的内部用词）；如有活动截止日期，建议用汉字简洁注明（如「2 月 25 日截止」）。

## 禁止（MUST NOT）

- **禁止**使用京东自研问卷系统以外的外部问卷系统录入。
- **禁止**题干标注的题型与实际设置不符（如标「多选」实为单选）。
- **禁止**将涉及用户隐私的题设为必答，或不注明隐私保护即采集隐私。
- **禁止**中奖公示泄露用户敏感信息（pin/昵称/联系方式/地址等）。
- **禁止**招募问卷的标题/文案让用户误以为「填问卷即等于接受访谈」。
- **禁止**问卷缺少欢迎语或结束语即对外推送。

## 检查清单

发布前逐项核对 📝：

- [ ] 1. 使用京东自研问卷系统（洞见/京东调研/京东良研）录入
- [ ] 2. 有欢迎语，且含主题 + 数据隐私条款 + 奖励信息；承诺与实际相符、奖励无歧义
- [ ] 3. 正文无错别字；字号 12–14 号；用语通俗无生僻
- [ ] 4. 题干题型与实际一致；限选/矩阵题标注条件清楚；跳转逻辑正确
- [ ] 5. 隐私题、非特殊开放题已设为非必答；隐私题题干已注明保护话术
- [ ] 6. 有结束语，并说明奖励发放方式 / 引导抽奖
- [ ] 7. 设置即时发奖；中奖公示已对敏感信息脱敏
- [ ] 8.（招募问卷）标题文案已注明「访谈报名/面对面/电话访谈」；题量与难度受控；甄别文案有亲和力
- [ ] 9. 投放文案结构完整（Push：标题+背景+利益点+入口；短信：签名+背景+利益点+截止+短链）；APP Push ≤45 字
- [ ] 10. 甄别题 1–3 个；甄别/提前结束文案亲和、可告知原因

## 例外与审批

- 问卷内容与投放物料**审核未通过不得对外推送**；审核由用户研究部内部进行。
- 确需偏离上述规则时，由问卷负责人在审核中说明理由并留痕。
- （推送数量、申请与轮值审批等属第一部分「消息盒子推送规范」的推送运营流程，不在本规范范围。）
