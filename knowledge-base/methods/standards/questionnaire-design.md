# 问卷设计规范

## 适用范围

- 适用于向用户投放、用以测量其网购行为或态度的各类**结构化问卷**（定量调查为主）。
- 覆盖问卷的**构成、题目与选项编写、设计流程、上线前自查**全过程。
- 问卷调查作为社会调查常见的研究方法，广泛应用于各类调查研究；规范的目的是保障用户填答质量与问卷数据的**信度、效度**。

### 背景：一份问卷的构成

问卷是一种通过书面形式与被调查者沟通的工具，通常包括以下几部分：

- **封面信**：一封致谢被调查者的短信，常放在问卷封面。一般需说明：我是谁（主办单位或调查者身份）、我们要调查什么（主要内容）、我们为什么进行这项调查（目的）、我们为什么找你（抽样标准与方法）、我们的调查有什么用（填答值不值得）、我们的调查不会有损于被调查者的利益。商业问卷为保障填答率常设置填答奖励，奖励细则同样需在封面信上标记清楚。
- **指导语**：告诉被调查者如何正确填答，包括回答说明、对某些问题涵义的进一步解释、对特殊或复杂填答形式的举例等。基本假设是被调查者可能从未见过、更没填过问卷。例如多选题注明最多可选几项；对「同城寄送」等特殊概念详细说明服务内容与可感知权益。
- **问题和答案**：在内容上可归为三大类——**特征问题**（测量基本情况，如年龄、性别）、**行为问题**（测量过去发生或正在进行的实际行为与事件，如「上个星期您打开几次京东 App」）、**态度问题**（测量对某事物的看法、意愿、态度、情感、认识等主观因素，如「依您的看法，好的快递服务应具备哪些要素？」）。
- **编码**：给问题和答案编上数码，以便用计算机进行统计处理和分析。现在常用的在线问卷系统在录入问题答案、导出数据时已自动完成编码步骤。

### 背景：好问卷的标准

- **具有较高的信效度**。信度即测量的稳定性和可靠性（对同一事物重复测量所得结果的一致性程度）；效度即一项测量真正测到了它所要测量事物的程度。
- **适合研究目的和内容**。看它是否符合研究目的要求、所含问题与研究内容的关系是否密切。
- **适合调查对象**。问卷是为被调查者「愿意看、也能够看」而设计的；样本构成复杂（职业、经济状况、文化程度、生活方式、心理状态有差别）时尤需注意。
- **问题少而精**。最常犯的毛病是问题太多太长——研究者想问的总比该问的多，贪多求全，到分析时却发现许多资料毫无用处，白白浪费人力物力和时间。

## 问卷结构与题序

- **固定模块顺序**：问候语 → 指导语 → 核心问题 → 人口统计学问题 → 结束语（缺一不可）。
- **核心问题排序**：先易后难、先一般后敏感、先封闭后开放，并按「认知 → 行为」逻辑排列。
- **敏感问题后置**：收入、隐私等敏感问题置于问卷后半段，降低抵触。

## 提示语设计规范

### 封面信

| 要素 | 字数上限 | 要求 |
|---|---|---|
| 问候语 | ≤10 字 | 使用敬语，表达尊重及亲切感 |
| 自我介绍 | ≤10 字 | 作为调研内容主语展示，无需特别介绍 |
| 调研内容 | ≤10 字 | 笼统表达行业方向，不泄露人群/产品话题（避免甄别失效） |
| 用户价值 | ≤30 字 | 强调被访者身份稀有/反馈指导产品提升 |
| 调研奖励 | — | 仅说明最高奖励形式和额度，具体每种奖励新开一行 |
| 调研时间 | — | 阿拉伯数字格式长日期 |
| 安全承诺 | ≤20 字 | 个人信息安全，填答内容最小范围使用 |
| 致谢语 | ≤10 字 | 简单致谢即可 |

**格式**：除问候语/致谢语/具体奖励外，其余内容 1 段内展示。问候语、调研内容做视觉加强。括号用英文半角，其余标点中文全角。

### 指导语

- 单一题型问卷：填答方式前置至提示语后展示
- 多题型问卷：每题题干中进行填答说明，不整体生成指导语
- 整体 ≤100 字，线框加粗/底色区域，首行顶格"填答说明："
- 每项内容单独一行，行首设项目符号，行内无句号，行尾无标点

## 题型分类与设计规则

### 题型分类

| 类别 | 题型 | 核心价值 | 核心风险 | 适用场景 |
|---|---|---|---|---|
| 封闭式 | 判断题 | 认知负荷极低，应答最快 | 信息量单薄，强迫非黑即白 | 人口学筛选、行为事实筛查 |
| | 单选题 ⭐优选 | 标准化极高，便于交叉分析 | 选项不当导致大量"其他" | 分类属性收集、频次判断 |
| | 矩阵单选题 | 节省版面，便于横向对比 | 惯性应答串行效应，移动端不友好 | 同一主体多维度评价（子项≤5） |
| | 打分题 | 定距数据可计算均值标准差 | 标准漂移，趋中偏差 | NPS、易用性快速打分 |
| | 量表题 | 语义锚点稳定测量态度 | 东亚中庸倾向，严格属定序 | 心理学态度、品牌形象、满意度 |
| | 多选题 | 全面覆盖行为全貌 | 分析复杂无法排序 | 生活习惯清单、购买决策因素 |
| | 矩阵多选题 | 极度节省问卷长度 | 认知灾难，**强烈建议不使用** | 仅限行≤3、选项≤4 |
| | 排序题 | 强制输出优先级关系 | 认知负荷极高，选项≤5 | 功能优先级、品牌偏好排位 |
| 开放式 | 单行文本题 | 获取精确事实 | 输入不规范需清洗 | 年龄、公司名、邮编 |
| | 多行文本题 | 质性深度，内容效度保障 | 拒答率>50%，编码成本高 | 预调查、NPS 低分原因、投诉建议 |

### 各题型题干规则

| 题型 | 题干必须 | 禁止 | 正例 | 反例 |
|---|---|---|---|---|
| **判断题** | 含二元锚点（是否/有无/正误） | 态度类陈述 | "您是否持有该银行的信用卡？" | "您认为这款手机好用吗？" |
| **单选题** | 含"最"或排他性指令 | 多选指令 | "最常使用的出行方式是什么？" | "您使用哪些出行方式？" |
| **矩阵单选** | 统领行与列的逻辑关系 | 混入不同类项 | "请对以下各方面满意度评价" | 行内混入"卫生"+"品牌知名度" |
| **打分题** | 明确最高分最低分锚定语义 | 无锚点纯数字 | "0分=非常不满意，10分=非常满意" | "请给这款APP打个分" |
| **量表题** | 陈述句（非疑问句） | 所有陈述指向正向 | "【我认为操作步骤繁琐】认同程度" | "请问您觉得复杂吗？" |
| **多选题** | 明确"选几项"或"可多选" | 缺乏限选指令 | "曾购买过哪些品牌？(限选3项)" | "您喜欢哪些品牌？" |
| **矩阵多选** | 加入串行效应警示，行数≤3 | 行数>3 | "请逐行判断，近期出现过哪些症状" | — |
| **排序题** | 明确排序依据和方向 | 方向不明 | "请按重要性从高到低(1为最高)排序" | "请对以下功能排序" |
| **单行文本** | 含格式示范 | 无单位/格式 | "您的年龄：__岁(请填阿拉伯数字)" | "您的年龄：" |
| **多行文本** | 收窄范围+引导性追问 | 太宽泛 | "打1分基于哪些具体事件？(不少于20字)" | "您的建议是什么？" |

### 答案设计规范

**通用规则：**
- 穷尽性与互斥性（MECE），无法穷尽时增加"其他(请说明)："
- 答案与题干维度一致，所答即所问
- 程度/等级语义枚举值数量为偶数更佳（避免骑墙）
- 单一选项 ≤10 字，默认字体字号，居中展示

**各题型答案专用规则：**
- 判断题：选项固定两类（是/否），等长无倾向
- 单选题/多选题：选项>7 个时改双列展示
- 矩阵单选/多选：所有行的选项列必须完全一致
- 打分题：0-10 连续正整数，两端极值赋予语义
- 量表题：按量表模型要求生成选项
- 排序题：每选项对应 1 个从"1"开始的连续正整数

### 题目组合顺序

1. **优先展示行为类**（客观事实易回忆）
2. **其次展示态度类**（意见评价）
3. **最后展示特征类**（人口统计，稳定属性）
4. 优先简单易答 → 优先调动兴趣 → 优先熟悉问题
5. 开放式问题集中放在所有封闭题之后、特征类之前

## 必须遵守（MUST）

> 以下为可硬性核对的题目/选项编写规则。

1. **一题只问一件事，禁止双重含义**。不得把多个主体/对象合并进一个题目，否则回答者难以选择、也得不到准确答案（如「您和您的家人使用过京东吗」——本人与家人的使用情况可能不一致）。
2. **题干必须中性、不得带倾向性或暗示**。不得用预设了立场的问法（如「您认为京东的快递费用是否应该降低呢？」带明显肯定倾向），应改为中性问法（如「您认为京东的快递费用如何」）。
3. **题型必须与测量目标匹配，且与选项协调**。要测「存在哪些问题」用多选题（选项为并列的问题项）；要测「对每一方面的态度程度」改为程度量表式问法（如「您认为京东在以下方面的表现如何」）。不得出现题干与选项答非所问的搭配。
   反例（题干「您认为京东目前存在的问题是？」搭配如下选项，会产出「……商品价格基本上可以」这类答非所问的组合）：

   ![](https://img12.360buyimg.com/ling/jfs/t1/180787/22/24901/16885/62870bacE83221ce0/14ed6dbb16b897cd.png)
4. **表示程度/频率的选项必须给出可量化、统一口径的锚点**。不得使用「经常/有时/很少」这类各人理解不一的模糊量级；应改为客观频次（如「请问您平常多久使用一次京东」→「一周一次、一周两次……」）。
5. **抽象概念必须先操作化为可回答的具体问题**。不得直接询问被调查者「属于哪个阶层」等专业/抽象词汇，应拆解为收入水平、消费习惯等行为层面的易答问题来间接衡量。
<!-- 合并自 Joyspace「定量问卷AI知识库」·孙文佳《定量问卷设计通用规范》，2026-06-25 -->
6. **禁用双重否定**。不得使用「您不认为该功能没有价值吗？」这类双重否定句式，应改为单一倾向明确的陈述问法。〔与 MUST 2「题干中性」配套的细化规则〕
7. **选项不得仅设单一倾向，须覆盖完整应答范围（避免极端效应）**。满意度等量表不可仅设「满意/非常满意」这类同向选项，须给出可覆盖负向到正向的对称区间。〔与 SHOULD「选项穷尽」一致，单一倾向禁用为新增细化；信效度门槛见下方「信效度控制」〕

## 推荐（SHOULD）

> 以下为提升填答质量与可读性的设计建议（经验性，非硬性）。

- **选项宜穷尽、互斥、且处于同一层次**。封闭题的答案最好覆盖所有可能取值（穷尽）、彼此不重叠（互斥）、不混用不同维度（同一层次）；答案不穷尽/不互斥/不同层次是常见的「答案设计不合理」缺点。
- **题干尽量口语化、通顺**。避免过度书面化（如「您是否是京东 plus 会员」宜改为「您是京东 plus 会员吗」）；检查语句本身是否通顺、有无前后重复的冗余（如「……能否提升……，或相反？」后半句多余）。
- **题干含义应明确具体，避免笼统**。如「您对京东的评价是？」未交代评价的对象（商品 / 促销 / 物流），应限定到具体方面。
- **避免「考试式」提法**。如「请您判断以下几种说法是否正确」会给回答者心理压力，宜改为「是否同意」。
- **封面信内容完整且简明**。既要呈现调查内容与目的（否则回答者心存疑问），又不啰嗦——能用一句话说清目的与内容就不展开。
<!-- 合并自 Joyspace「定量问卷AI知识库」·孙文佳《定量问卷设计通用规范》，2026-06-25 -->
- **题干宜用简单陈述句，单句字数尽量 ≤ 20 字**（量化口径，上线前可核对，见检查清单）。
- **人群语言适配**：老年用户禁用网络用语；B 端用户可适当使用行业通用术语。示例：「你觉得京东的物流配送速度大概还可以吗？」宜改为「你对京东的物流配送速度是否满意？」
- **题量与时长控制（量化）**：单份问卷题量 15–25 题、应答时长 ≤ 8 分钟为宜，最长不超过 30 题（≤ 10 分钟），超量需拆分或优化。
- **必答题数量 ≤ 80%（量化）**；开放题仅保留 1–2 个核心问题，避免应答负担过重。
- **每个题型前标注清晰指导语**，如「可多选」「请选择最符合的一项」「1 分代表非常不满意，5 分代表非常满意」。
- **敏感问题不可强制应答**，需设置「不便透露」「其他」等兜底选项（与抽样/伦理要求呼应）。

### 推荐的设计步骤（5 步）

1. **准备性工作**：首先明确调查总体目标——问卷主要目标是什么？研究假设与关键变量是什么？据此应寻求什么资料、希望以什么信息作为结果？研究类型是描述性还是解释性？其次，设计前最好先对调查对象基本情况有所认识，以便更好地设计问题形式、提问方式与用语。
2. **探索性工作**：常见方式是深入社会生活背景做一段时间非结构式访问，与各类回答者交谈、提问，把各种设想拿去尝试比较，初步获得对问题想法、实际语言、可能回答种类的印象。
3. **设计问卷初稿**：常用两法——**卡片法**（把每个想到的问题写在卡片上，再分类、按逻辑结构排序）与**框图法**（先据研究假设与资料逻辑画出问卷结构、排好前后顺序，再思考每部分设计什么问题）。
4. **试用与修改**：**客观检查法**——发布到正式调查总体，回收 20–50 份，看是否某部分填答率很低或某些题答案过于集中；**主观评价法**——请调研同行试答、做专业问卷审核并提修改意见。
5. **问卷的录入**：在线问卷系统已替调查者考虑了部分注意事项；录入时自查——是否用不同字号字体区分问题与答案、题量大时是否分页并列小标题、是否标记了需额外注意的重点，并认真校对避免错字。

## 信效度控制

<!-- 合并自 Joyspace「定量问卷AI知识库」·孙文佳《定量问卷设计通用规范》，2026-06-25 -->

> 量化的信效度门槛（现有规范在「好问卷的标准」中定义了信度/效度概念，本节补充可核对的量化阈值）。

- **同一测量维度题项数量 ≥ 3 个（量化）**，题项间需具备内部一致性，**Cronbach's α ≥ 0.7 为合格**。
- **核心维度优先采用成熟量表**（如满意度用 SCS 量表、NPS 用标准量表），以保障内容效度。
- **避免极端效应**：选项区间需覆盖所有可能应答范围，禁用单一倾向选项（与 MUST 第 7 条一致）。

## 禁止（MUST NOT）

> 九类常见错误中可直接判错的禁止项（与上方 MUST 互为正反）。

- **禁止**一题包含双重含义（多个主体/对象合并）。
- **禁止**前后重复、自相缠绕的题干。
- **禁止**带倾向性、诱导性、预设立场的问法。
- **禁止**题型与测量目标错配、题干与选项不协调（答非所问）。
- **禁止**用未定义口径的模糊程度/频率词（经常/有时/很少）替代可量化锚点。
- **禁止**直接抛出抽象/专业概念让用户自行归类（如「您属于哪个阶层」）。

## 检查清单

上线前逐项核对 📝：

- [ ] 1. 封面信已说明：主办方身份、调查内容、目的、抽样理由、价值、不损害被调查者利益；商业问卷已写明奖励细则
- [ ] 2. 指导语已覆盖填答说明、特殊概念解释、复杂题型举例（如多选题最多选几项）
- [ ] 3. 每道题只问一件事，无双重含义
- [ ] 4. 题干中性、无倾向性/诱导性，无「是否正确」式考试提法
- [ ] 5. 题干含义具体明确，无笼统宽泛的对象
- [ ] 6. 抽象概念已操作化为可回答的具体问题
- [ ] 7. 选项穷尽、互斥、同一层次；题型与测量目标匹配、与选项协调
- [ ] 8. 程度/频率类选项有统一、可量化的锚点
- [ ] 9. 语言口语化、通顺，无书面化拗口或冗余表述
- [ ] 10. 问题少而精，与研究目的强相关；无可有可无的题
- [ ] 11. 已用字号字体区分题目与答案，题量大时分页并加小标题、标注重点
- [ ] 12. 已小样本试用（客观检查 20–50 份 / 同行主观评审）并据反馈修改
- [ ] 13. 已认真校对，无错别字
<!-- 合并自 Joyspace「定量问卷AI知识库」·孙文佳《定量问卷设计通用规范》，2026-06-25 -->
- [ ] 14. 题干为简单陈述句、单句字数尽量 ≤ 20 字；无双重否定
- [ ] 15. 选项无单一倾向，覆盖完整应答范围（避免极端效应）
- [ ] 16. 同一维度题项 ≥ 3 个；核心维度优先用成熟量表（信效度门槛 Cronbach's α ≥ 0.7）
- [ ] 17. 题量 15–25 题、应答时长 ≤ 8 分钟（不超 30 题 / 10 分钟）；必答题 ≤ 80%；开放题 1–2 个
- [ ] 18. 每个题型前已标注指导语；敏感题已设「不便透露/其他」兜底、不强制应答

## 例外与审批

- 本规范的「必须遵守」项为通用底线，**探索性/定性问卷**（以开放题替代访谈、前瞻探索）在题型与穷尽性上可有不同取舍，参见 `methods/toolbox/collection/qualitative-questionnaire.md`。
- 量表类题目的信效度、维度与计分另有专门方法，参见 `methods/toolbox/collection/scale-development.md`。
- 确需偏离上述规则时，由问卷负责人在评审中说明理由并留痕。
