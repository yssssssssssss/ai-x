# 调研报告撰写规范

> 来源：神灯圈子·吉国杰《用研新人成长记——调研报告撰写心得》原文 http://xingyun.jd.com/shendeng/article/detail/46546 （原作者伊萌，写于 2021-09-07）
>
> 内部参考规范。本规范在本库**单一出处**，其他文件引用而非复制。
> 落点：`methods/standards/`。本规范来源为「从 0 到 1 写一份用研报告（PPT 格式）」的新人经验沉淀，通篇为"建议/记得…哦/仅供参考"的经验口吻、未确立任何硬性规则；按"只把能确立的硬约束列为 MUST"的原则，本规范暂无 MUST 项，以**推荐（SHOULD）+ 检查清单**为主干。

## 适用范围

- 用户研究项目交付的**调研报告**（默认 PPT 格式，亦适用于其他文档形态的研究报告）。
- 覆盖报告生命周期三阶段：**报告撰写前**（准备）、**报告撰写中**（结构与论证）、**报告撰写后**（复核）。
- 一份好的用研报告应满足三层标准：
  - 基础：结构完整、逻辑清晰，能把事实阐述清楚；
  - 进阶：论证充分、材料丰富，有理有据地呈现调研结论；
  - 出彩：具有长期价值，不仅提供阶段性建议，也能为长期的产品业务规划提供参考。

## 必须遵守（MUST）

> 本规范来源为单一作者的新人成长心得，通篇以"建议/记得…哦/仅供参考"的经验口吻给出，未确立任何"必须/不得"级别的硬性规则。因此本节暂不列硬约束；下列要点均以**推荐（SHOULD）**呈现，请结合项目实际与团队约定执行。

## 推荐（SHOULD）

### 几条最该记住的要点

- **撰写前尽量让素材可直接应用。** 建议先把所需调研素材（问卷分析结果、深访记录表、桌面研究资料、执行过程照片等）规范命名、放到能轻松找到的地方，并确认内容完整、可直接使用——毕竟你也不想写着写着报告，再回头去查访谈原始录音或问卷原始数据。
- **主体尽量每页只讲一个主题/结论/观点。** 记得每页只承载一个主题哦，别在同一页堆叠多个结论。
- **论点别忘了配论据。** 主体部分每一页在给出结论时，记得一并呈现支撑该结论的论据（问卷数据、深访原话、田野观察、文献资料等），向需求方清楚说明"结论从何而来、为何得出"，让对方知道这个结论有理有据。
- **完成后建议复核再交付。** 报告写完后，建议至少做两件事：①通读一遍，检查错别字与语句是否通顺；②把业务方原始需求拿出来逐条对照，看是否有遗漏、是否清楚全面地回答了原始诉求——毕竟你也不想汇报时才发现犯了低级错误。

### 报告撰写前（准备）

- **备好 PPT 模板，而非仅套用网络模板。** 根据报告主题选定配色（一般不超过 2 个色系）；提前做好封面、封底、目录、章节标题页；为主体内容页添加母版、打好标尺、附上页码。

  ![](https://img20.360buyimg.com/ling/jfs/t1/190464/36/8575/27303/60c9c750Ec0a3f0fb/490841ce27fc7e3d.png)

- **做好时间安排，预留整块时间。** 报告撰写需要整块时间以保证注意力集中、产出持续。若有其他项目并行，建议先清掉琐碎工作、再留整块时间给报告（属个人工作习惯，供参考）。
- **与领导确认交付 DDL，并自拆每日 DDL。** 可细化到每天完成几页；不必按自然页序，按内容难易分配权重（如某天写了 2 页难的，剩余配额挑简单内容补足，确保当日总数达成即可）。
- **克服完美主义。** 不指望一次搞定；先出初版再不断迭代，比追求一步到位更有针对性，也更利于按时交差。

### 报告撰写中（结构与论证）

- **先搭"六股"结构再填内容。** 用研报告可参考如下通用结构：

  | 章节 | 建议页数 | 说明 |
  |---|---|---|
  | 项目背景 | 1 页 | 交代该项目需求的背景 |
  | 执行概要 | 1 页 | 呈现该项目使用的方法 |
  | 研究内容 | 1 页 | 呈现该项目研究的内容 |
  | 主体部分 | 若干页 | 每一章节可用 1 页做小结汇总 |
  | 整体结论 | 1–3 页 | 汇总整个报告的结论 / 策略建议 |
  | 附录 | 若干页 | 问卷结构、深访名单等 |

- **先在模板每页写上"这一页要表达什么"的提示。** 即便主体部分复杂，也建议逐页标记待写内容（描述不精确、自己看懂即可）。好处：清楚 PPT 结构与各部分页数、便于排每日 DDL、能据标记判断各页完成难易、并发现哪部分素材有欠缺需补充。

  ![](https://img14.360buyimg.com/ling/jfs/t1/192180/13/8547/276041/60c9d0deE9483302f/ffc4a4ff2c485b51.png)

  ![](https://img12.360buyimg.com/ling/jfs/t1/185475/6/9179/269096/60c9d0b3E7859dfc4/e6f4fcfe306dd5a3.png)

  ![](https://img14.360buyimg.com/ling/jfs/t1/188318/11/8739/1247089/60c9cd3dE139bd617/d5c3b041216a113c.png)

- **重在事实，高在洞察。** 把调研结论、用户反馈描述清楚是基本准则；在事实清楚之上再加洞察，更能体现用研专业能力。例：分析京东直播在京东 App 的入口时，呈现各入口及对应使用场景是基本要求；进一步可发现当前入口主要与购物黄金流程结合，而闲逛、内容唤醒场景较少，与站内频道及营销场景的合作仍有待挖掘——结论由此更深入、立体。

  ![](https://img14.360buyimg.com/ling/jfs/t1/179936/2/9500/243768/60c9ca18Ec5645a10/d92c12ae7c56a4eb.png)

- **善用文献等二手资料补强洞察。** 当难以独立总结洞察、或不知从何角度分析时，应学会求助：一方面请教有经验的同事，另一方面借助二手资料。例：分析购物直播用户需求趋势时，访谈仅能看到不同用户对直播的定位差异（选品参考 / 因主播有趣而看），尚不能断定趋势；查阅文献发现购物直播朝多样化发展（商品、主播类别、直播形式多样），结合一手资料即可综合得出需求发展趋势。

  ![](https://img20.360buyimg.com/ling/jfs/t1/196313/39/8471/230067/60c9ca88Effd9ba29/e22c4434a16431ba.png)

- **排版服务于阅读，不追求复杂设计。** 好的排版要清晰、可视化：让读者快速找到核心结论、理解内容间逻辑关系、获得轻松的阅读体验。具体应遵循字号段落统一、框线粗细协调、分布均匀对齐；并运用图示、图表代替大段文字，辅助读者理清逻辑关系。

### 报告撰写后（复核）

- **检验能否串联成一条主线。** 试着自己把报告讲一遍，看各部分能否流畅串联成主线。主线无固定要求，可按用户使用产品的流程串联（触达方式—使用场景—使用体验），也可按业务策略串联（拉新—留存—促活）；再据此检查各部分衔接逻辑，删冗补缺。

## 几点提醒（避免）

> 源文未把以下情形表述为"禁止"，这里作为**推荐（SHOULD）**层面的提醒列出。

- **尽量避免只写论点而不给论据。** 仅有结论、缺乏数据 / 原话 / 观察 / 文献支撑的页面，最好在交付前补齐论据。
- **尽量避免一页堆叠多个主题/结论。**
- **尽量避免跳过复核直接汇报。** 错别字、语句不通、遗漏原始需求等低级问题，建议在汇报前先排除。

## 检查清单

撰写前：
- [ ] 调研素材（问卷结果 / 深访记录 / 桌面研究 / 执行照片等）已规范命名、可轻松找到、内容完整可直接用
- [ ] PPT 模板就绪：配色（≤2 色系）、封面 / 封底 / 目录 / 章节页、母版 / 标尺 / 页码
- [ ] 已与领导确认交付 DDL，并拆解出每日页数配额

撰写中：
- [ ] 已先搭好"六股"结构（项目背景 / 执行概要 / 研究内容 / 主体 / 整体结论 / 附录）
- [ ] 每页已标注"这一页要表达什么"
- [ ] 主体每页只讲一个主题/结论
- [ ] 每个结论都配有论据，并交代了结论来源与推导
- [ ] 在事实之上尝试给出洞察（必要时借助同事经验与二手文献）
- [ ] 排版：字号段落统一、框线协调、对齐均匀，图示/图表替代大段文字

撰写后：
- [ ] 通读全文，无错别字、无不通顺语句
- [ ] 对照业务方原始需求逐条核对，无遗漏、已清楚全面回应原始诉求
- [ ] 自讲一遍可串联成一条主线，各部分衔接逻辑顺畅

## 例外与审批

- 本规范源自单一作者经验沉淀，**时间安排、配色、按难易拆页**等属个人工作习惯，可结合项目实际与团队约定调整。
- 非 PPT 形态的研究报告（如长文档），可在保持"结构完整、论点配论据、完成后复核"三条核心约束的前提下，灵活调整章节与排版形式。
