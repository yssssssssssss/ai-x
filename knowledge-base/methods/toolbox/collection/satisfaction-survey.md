# 满意度问卷设计与数据清洗

> 来源：Joyspace《满意度问卷设计与数据清洗》 https://joyspace.jd.com/pages/0ABrbCeAe0AVQ3kqsk0A
> 部分内容并入自：Joyspace《NSS介绍：用户满意度下降时如何归因》 https://joyspace.jd.com/pages/2rD0fd6djkqgWUZjNHwa

> 一句话：用结构化的量表问卷量化用户对新版本的满意度与具体体验，并通过严格的数据清洗剔除无效作答，让"满意度提升了多少"成为可信、可比、可决策的数字。
> 落点：`methods/toolbox/collection/`。方法本体在此**唯一出处**，scenarios/skills 引用而非复制。

满意度问卷的本质是一个**测量工具**，它的质量由两条链路决定：**设计链路**保证"测得准"（效度），**清洗链路**保证"答得真"（信度与有效性）。整套方法围绕明确的测量构念（满意度、易用性、推荐意愿等）设计标准化量表问卷，并对回收数据做去噪、校验与质量评估。核心逻辑是：先把抽象概念操作化为可测题目（构念→维度→题项），用规范量表采集，再用统计手段确认"测得准（效度）、测得稳（信度）、答得真（清洗）"。

## 何时用 / 不适合

- **适用**：
  - 版本上线后做满意度/体验效果评估，回答"这次迭代是否提升了用户的主观体验、提升了多少、是否达到预期"。
  - 定期满意度追踪：按固定周期重复同一问卷以观察趋势。
  - A/B 或灰度版本主观体验对比：对不同版本人群投放同一问卷比较主观指标。
  - 为驱动因素分析与优先级排序提供量化输入。
- **不适用 / 局限**：
  - 只反映"被问到的"主观态度，受题目质量与样本代表性强约束。
  - 问卷只给"是什么 / 多少"，**无法解释"为什么"**，归因需配合定性方法。

**在产品迭代期的定位**：满意度问卷属于迭代期用研的【效果评估】大类，采集载体以**问卷**为主，常在版本上线后配合 VOC（用户之声）采集同步开展。其输出的量化结果是后续分析的"原料"，而非终点。最佳使用时机有三个：

1. **版本上线 2–4 周后**：用户已形成稳定使用习惯，适合做一次满意度效果评估。
2. **长期追踪**：按固定周期重复同一问卷以观察趋势，此时务必保持题目与题序一致以保证可比。
3. **A/B 或灰度对比**：对不同版本人群投放同一问卷，比较主观指标差异。

## 前置准备

- **招募与样本**：界定目标人群与抽样框，设置配额保证代表性，规划邀请方式与回收目标。在线 opt-in 样本误差约为概率样本的两倍（Pew），抽样代表性需重点把控。
- **测量目标与构念**：写清要测的概念、对应假设和决策用途，确定核心指标（CSAT / NPS / CES / NSS）。
- **物料**：构念定义、历史问卷题库、标准化量表、问卷投放与回收工具、预测试（pilot）方案。

**触发时机原则（来自 NSS 实践）**：在用户**深度体验产品功能后**再触发问卷，确保反馈来自真实使用而非无效填答。

## 执行步骤

1. **定义测量目标与构念**：写清要测的概念、对应假设和决策用途，确定核心指标（CSAT / NPS / CES / NSS）。
2. **构念操作化与选量表**：把构念拆成维度与题项，优先复用成熟标准化量表，选定点数与标签。
3. **撰写题目与排序**：逐题检查引导性、双重问题、双重否定、术语；保证选项穷尽互斥；按主题逻辑排序，人口统计后置。
4. **设计抽样与配额**：界定目标人群与抽样框，设置配额保证代表性，规划邀请方式与回收目标。
5. **预测试（pilot）**：小样本试投，检查理解歧义、答题时长、跳转逻辑与初步信度。
6. **正式投放与回收**：监控回收率与样本结构，必要时追投以缓解无应答偏差。
7. **数据清洗**：去重 → 剔 speeders → 查 straight-lining → 过 attention check → 逻辑校验 → 处理缺失 / 异常值。
8. **信效度与指标计算**：算 Cronbach's α、各维度得分与核心指标，输出报告并交付下游分析。

### 量表与计分口径

- **Likert 量表**：由心理学家 Rensis Likert 于 1930 年代提出，用"非常不同意—非常同意"测量对陈述句的赞同程度，单题称为 "Likert item"，常用 5 点，也可用 7 点或 9 点（NN/g）。点数越多分辨力越高但认知负担越大；电话或移动端宜用更少选项。
- **CSAT**：多为 5 点满意度量表。计分口径 **CSAT = (4–5 分高分数量) / 总填答量的占比**。
- **NPS**：用 0–10 单题"向朋友 / 同事推荐的可能性"，9–10 为推荐者、7–8 为中立者、0–6 为贬损者。**NPS = 推荐者占比 − 贬损者占比**（维基百科）。
- **NSS（净满意度）**：**NSS = %满意者(9–10 分) − %不满意者(0–6 分)**。
- **CES（费力度）**：常用 1–7 量表测量"完成任务所需的努力"，**CES = 所有填答得分的平均值**（或分析低分比例）。

> 选型速记：优化**具体功能 / 流程**用 **CES**；评估**单次服务质量**或为产品做**全面体检**用 **CSAT** 或 **NSS**；衡量**忠诚度 / 口碑、预测增长**用 **NPS**。

### 经典三段式问卷（来自 NSS 实践）

为"最小化用户投入、最大化可执行洞察"，一份满意度问卷可采用经典三段式结构（建议用户在 30 秒内完成、问题数控制在 3 个以内）：

1. **核心计分（定量）**：「请您对 XX 平台「XXX 模块」使用感受打分」（必填）。0–10 分（非常不满意—非常满意）数字评分量表，直接测量各系统 / 模块体验分值（与上文 NSS 计分口径对齐）。
2. **反馈归因（定性）**：「您对 XX 平台还有哪些改进建议，欢迎反馈」（选填）。对**所有用户**开放的文本框，给用户反馈出口，获取额外洞察及建议。
3. **二级归因（定量）**：「您认为有哪些不满意或需要改进的地方？」（多选，必填）。**仅对 0–6 分低分用户**进一步提供不满意原因选项，将低分原因归类，通过占比分析快速定位核心原因，并转为后续"可执行"动作。因此选项设计必须**具体、可操作、覆盖核心场景**，避免后期分析无抓手。

## 关键技巧与注意

**题目编写自检清单（写完每题逐条过）**

| 检查项 | 不合格示例 | 修正方向 |
| --- | --- | --- |
| 无引导/情绪词 | "你有多喜欢这个超赞的新功能？" | "你对这个新功能的满意度是？" |
| 非双重问题 | "速度和界面是否满意？" | 拆成"速度满意度""界面满意度"两题 |
| 无双重否定 | "你不认为它不好用吗？" | "你认为它好用吗？" |
| 无行话术语 | "对 onboarding 流程满意吗？" | "对新手引导流程满意吗？" |
| 选项穷尽互斥 | 年龄段"18-25/25-30" 重叠 | 改"18-24/25-34"且含"其他/不愿透露" |
| 量表对称 | 4 正 1 负的偏态选项 | 正负对称，必要时加"不适用" |

**题目编写要点**

- 措辞的细微变化就能显著改变结果（Pew），题目应清晰、具体、可回答；闭合选项必须**穷尽且互斥**；避免双重否定、行话与生僻缩写，警惕带情绪色彩的词。
- 坚决避免 **double-barreled（双重问题）**——把两件事塞进一题（如"本次更新的速度和界面是否令你满意？"），SurveyMonkey 与 Pew 都建议拆成两题。
- **题序与锚定效应**：先答的题会影响后答的题，存在"对比效应"与"同化效应"，问卷应按主题逻辑推进、以吸引人的题开场、把人口统计题放在最后；选项顺序会引发电话场景的"近因效应"和自填场景的"首因效应"，故 Pew 常随机化选项顺序。

**不满意原因选项设计三法（来自 NSS 实践）**

二级归因多选题的选项质量直接决定后期分析能否落地，可用以下三种方式设计：

1. **通用选项库**：拿来即用、前期设计成本低；弊端是选项笼统、后续分析成本高，仍需进一步挖掘痛点。常用类目：
   - **产品功能类**：当前功能不好用（请说明）、缺乏所需功能（请说明）
   - **系统性能类**：系统卡顿/加载慢、系统异常/崩溃
   - **界面设计类**：视觉布局复杂/混乱
   - **引导帮助类**：缺乏功能指引/操作说明
   - **业务规则类**：平台规则不合理（如审核规则、封禁规则等）
   - **其他**：请说明（覆盖未预料到的情况）
2. **按操作流程节点设计**：沿用户操作链路拆分选项。如视频发布模块可设为：封面/标题、视频上传、商品选择、话题添加等，从而较精准定位到具体使用体验较差的功能，指引优化方向。
3. **迭代问卷本身**：定期回顾反馈内容，若发现大量用户用自己语言描述同一个此前没注意到的问题（如创服平台反馈中集中提到"账号封禁申诉"），就把它**作为一个新选项**加入下一版问卷，让归因更精准。

## 常见坑

- **跳过构念定义直接抄题**：题库东拼西凑，测出的指标无法对应任何决策。
- **盲目堆量表点数或乱改题序追求好看**：破坏可比性，让追踪数据前后不可对照。
- **double-barreled 与引导性措辞**：把两件事或预设立场塞进一题，污染数据源头。
- **只看回收率、不查应答质量**：高回收率不等于真实——研究表明回收率是无应答偏差的弱预测因子（参与偏差），且 speeders 与机器人会拉高"伪回收"。
- **把 Cronbach's α 当万能体检**：高 α 不证明单维度，"删题提 α"可能虚高；应先验证维度结构。
- **用问卷强行解释"为什么"**：量表只给"是什么 / 多少"，归因需配合定性方法。

## 产出与交付

- **直接产出**：清洗后的有效样本数据集、CSAT / NPS / NSS / CES 等核心指标、各维度满意度得分、信效度报告。

**数据清洗规则（伪代码 / SQL 思路）**

```sql
-- 1) 去重：同一用户仅保留最早一次有效作答
ROW_NUMBER() OVER (PARTITION BY user_id ORDER BY submit_time) = 1

-- 2) speeders：答题时长 < 题量预估时长的 1/3 即标记
WHERE duration_sec < expected_sec / 3

-- 3) straight-lining：所有量表题方差为 0（全选同一档）
WHERE VAR(scale_items) = 0

-- 4) attention check：陷阱题未按指定答案作答
WHERE trap_q <> expected_trap_answer

-- 5) 逻辑一致性：互斥/矛盾作答
WHERE (never_used = 1 AND usage_freq = 'daily')

-- 满足以上任一 → flag=invalid，从分析集中剔除
```

缺失值处理：题级缺失少量可成对删除或均值 / 中位数填补，整段空白则按无效样本剔除，并记录缺失机制（随机缺失 MCAR/MAR 与否）。

**SPSS 实操清洗经验（补充上文 SQL 思路在工具与流程上的落地）**

- **工具分工**：基础描述统计与交叉分析 Excel 与 SPSS 均可实现；实践中常**先用 SPSS 做清洗与分析**（双视图查看方便、可写语法批量处理），**再导出 Excel 编辑图表格式**（可视化更丰富、改外观便捷）。
- **数据视图规范化（SPSS 特有，前置一步）**：逐列调整变量视图 11 列以减少后续分析 / 写语法的失误概率——名称（与原始问卷编码一一核对）、类型（选择题为数字、填空题为字符串）、宽度（同类题型对齐）、小数（选择题为零）、标签（直观描述并把名称并入标签）、值（与原始选项核对）、缺失（逻辑跳转题的"未选择"会被计为零值、不利交叉分析，须按原问卷跳转设置剔除该零值；缺失处理因问卷系统而异，京东以 limesurvey 为例）、列宽、对齐（数字右对齐、字符串左对齐）、测量（定类设名义、定序设有序、定距定比设标度）、角色（默认输入即自变量）。
- **清理无效样本的两条原则**：**从整体到部分、从一维到二维**。
  - **整体处理**：① 按填答完整性（剔未完整填答必答题的样本；非必答的敏感题视项目需求决定是否剔）；② 按提交时间（剔除早于 / 晚于投放计划时间的样本，如测试填答）；③ 按填答时长（剔过短——经验最低时长≈5 秒 × 题量——与过长（一般 > 30 分钟）样本）。
  - **按问卷三部分逐部分处理**：**甄别部分**剔不符合甄别条件者；**主体部分**剔连续性回答（同一选项过多，如 30 题选 25 次 A；或呈 "A-B-A-B" 规律）、剔违反固定填答逻辑（互斥题矛盾）、剔未通过陷阱题者；**属性部分**做属性题组内部（如 20 岁以下不应已婚、个人月收入不应大于家庭月收入）与题组间（如 < 18 岁一般无本硕博学历）的矛盾校验。
  - **各部分之间**再两两比较剔矛盾样本，比较时以题目少的部分对题目多的部分。
- **特殊题型（文本题）处理**：① **回码**——文本填答可量化或可并入既有选项时，转成数值并删除原文本（如用户在"其他，请注明"里填了已存在的"京东"选项，应回码并入京东）；② **重新编码**——不可回码的新答案另行编码并记入编码簿。

**频数 / 描述 / 交叉分析要点（补充计分口径之外的基础分析维度）**

- **频数分析**：**总体频数**——各题选项按降序排列，先看整体分布趋势，了解用户总体行为 / 态度偏好；**分组频数**——把相似选项合并成更宏观维度（如把"商品关注 11 要素"归为商品层面 / 平台层面），各选项百分比加总后对比维度，并可在维度内再降序排列。
- **描述性分析**：针对数值型单变量统计——集中趋势（均值、中位数、众数、百分位数）、离散程度（方差、标准差、均值标准差、极差、离散系数）、分布形态（偏度、峰度），为后续复杂分析建模铺垫。
- **交叉分析**：用户属性 × 行为 / 态度交叉，**关注显著高于 / 低于总体的格子并做标记**；当属性为定序变量时看行变量是否随升 / 降序呈趋势（如随年龄递增越关注品牌）；**注意：当某格样本量少于 30 时数据差异误差可能较大，应对照样本量做检验**。
- **数据格式规范化（交付前）**：① 必须**标记样本量**（每题总填答人数、每选项填答人数逐一标注）；② 形成**列联表**（题目 / 选项置行、样本属性置列）；③ 按题组**拆分 sheet** 便于查找。

**信效度**：

- **信度**用 Cronbach's α 衡量同一构念下多题的内部一致性（业界常以 0.7 为经验门槛），但维基百科提醒：高 α 不等于单维度，应先验证单维度再算 α，且"删题提升 α"可能造成 α 虚高。
- **效度**关注"是否真的测到了想测的构念"，靠构念设计、专家评审与相关性验证来保障。

**满意度调研项目通用流程（指标→问卷→投放→清洗→权重→算分）**

把上文的设计与清洗放进一条端到端的满意度项目流水线，可按以下六段执行（适用于版本 / 主站满意度评估与周期监测）：

1. **指标确定**：按产品 / 服务逻辑归纳一、二、三级多级指标（低级指标须穷尽高级指标覆盖的内容），再按四原则**筛选**——研究**重要性**（用户视角重要）、研究**可测量**（能统计计算）、策略**前瞻性**（长期有效、便于趋势对比）、策略**可操作**（指向可优化方向）。指标过多会推高填答成本、拉低数据质量。
2. **问卷设计**：设甄别条件（如近三个月了解 / 用过目标产品；站内投放且默认了解可省）；设配额条件使样本属性分布贴合整体画像（若计划事后加权可不设配额）；指标话术简练、用用户语言、关键词可标红；选项可选**梯级**（心理学梯级理论，7 级可压成 5 级）或**量表**（5 / 10 分制，题干与选项都标明分值语义）；背景信息与自有用户库填写规则保持一致。
3. **投放执行**：投放时间显著影响回收率（节假日 / 大促回收率可低至常规的约 1/10，周期监测应缩短回收间隔、提高频次）；按甄别条件以站内订单行为圈选用户、圈选时间贴近投放日；设置用户奖励；问卷上传后预览测试（注意终端显示、抽奖、用户名记录以免客诉）；申请渠道、正式投放（可延长、尽量不提前终止）；回收时原始数据增设"投放时间"列便于合并拆分；后台匹配用户属性为后续权重计算供数（属性维度越多权重越精准）。
4. **数据清洗**：分**问卷清洗**（完整性 + 有效性，有效性看填答时间 / 逻辑冲突 / 连续回答；SPSS 可在每步语法后加描述统计便于回溯，常用 `DESCRIPTIVES VARIABLES`、`SELECT IF`、`COMPUTE`）、**后台清洗**（按属性枚举值剔除越界 / 大量缺失样本，并对属性做合并等调整）、**权重清洗**（权重计算后，删除无法匹配权重的样本）。
5. **指标体系与权重计算**：将指标放入树状结构并赋权重——**专家评定**（主观，相关方讨论，重大调整需重定）或**主成分分析**（客观，SPSS"分析→降维→因子分析"，由三级→二级→一级→整体逐层加权；满意度分值在固定正区间，清洗后数据可直接计算，无需再标准化）。加权目的是用权重修正样本结构与整体用户结构的差异：样本权重 `fn = 目标样本量占比 / 实际样本量占比 =（后台某类型用户量 / 后台总用户量）/（该类型问卷样本量 / 问卷总样本量）`，可用 `COUNTIF`/`COUNT`/`VLOOKUP`/`ISNUMBER` 等函数实现。指标权重在保证标准化与效率时应尽量长期固定。
6. **分数计算**：先做**分数转化**——用"极差标准化—区间缩放法"把 5 / 10 分制折算为百分制：`百分制 =（X − min）/（max − min）× 100`（即五分制 `=（得分−1）/（5−1）×100`、十分制 `=（得分−1）/（10−1）×100`）；再算**样本得分**（一级指标得分 = Σ 各二级指标得分 × 对应权重）与**整体得分**（按各样本权重加权平均：`指标得分 = x1f1 + x2f2 + … + xnfn`）。
