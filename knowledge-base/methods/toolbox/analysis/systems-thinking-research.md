# 系统思维做业务调研

> 来源：神灯圈子·乔露《如何系统化地进行UBP业务调研》 原文 http://xingyun.jd.com/shendeng/article/detail/46583
> （原文作者关斯斯，写于 2022.01.25）

> 一句话：用系统思维（目标-变量-连接）跳出"表象陷阱"，借助因果回路图（CLD）从全局视角拆解 B 端业务流，搭建"调研-监测"体系。

## 何时用
- 适合的问题类型：
  - B 端/业务工具产品线（如 UBP 营销工具）需要从全局业务流出发做系统性调研，而非只盯单点反馈。
  - 用户/业务反馈信息冗杂、需要溯源问题本质与根因，避免"见树不见林""头疼医头脚疼医脚"。
  - 需要梳理多角色、多模块、多触点之间的内在联系，搭建持续运行的"调研-监测"体系框架。
- 与近似技术的区别：
  - 系统思维与 5W1H、六顶思考帽一样属于一种**思维方式/思维工具**，强调站在整体、动态的角度由浅入深看本质。
  - 系统由三大核心部分构成：**目标**（系统想实现的目标/想解决的问题）、**变量**（构成部分，也是实现目标的驱动力，可具象可抽象）、**连接**（变量之间或变量与目标之间的关联关系）。
  - 区别于只关注单个"变量"的局部分析：单个变量最容易被注意到，但"连接"和"目标"最容易被忽视，过度关注单个变量会让人局限、迷失目标。

## 输入数据要求
- 围绕一个明确的研究对象/业务流（如某条营销工具产品线、某场 618 大促活动）。
- 多渠道获取的业务与用户信息：
  - 深度访谈（理解用户、溯源"为什么会出现"和"怎么解决"）。
  - 小范围团队讨论 / 共创会（就最重要因素拉齐认知、达成共识，确定系统边界）。
  - 田野观察 / 录屏日志等方式获取关键人（who）与关键事（what）的信息。
- 涉及的全量角色对象、系统工具、任务场景等盘点清单（如营销运营、采销、商家等角色）。

## 分析步骤
> 总体逻辑：理清业务 → 确定研究边界识别关键变量 → 归类共性诉求 → 拆解目标落地。围绕"营销活动运营流程提效"提出目标愿景——建设一个覆盖全场景/全流程/全角色的一站式营销平台。

1. **理清业务，打好地基，助于共情**（盘变量）。带着以下问题思考：
   - 目标：研究什么？如关键事件/关键成果是什么、要解决什么问题。
   - 变量：组织部分/关联元素有哪些？如对象、产品（系统工具）、场景、任务等。
   - 连接：变量/元素间的交互关系是什么？如组织架构、信息流传递等。
   - 具象化示例：假设做一场 618 大促活动，先关注 3 个 W 和 1 个 H——找对的人（who，活动发起者/参与者）、找对的事（what，做什么/关注什么），通过田野观察/录屏日志等方式获取信息。
2. **理解用户、溯源问题**（确定研究边界，识别关键变量）。仍以"618 大促活动"为例，从大部分用户访谈中关注"为什么会出现"和"怎么解决"的问题。
3. **归类问题、梳理共性诉求**。访谈资料常存在信息冗杂，需从不同的场景环节、产品能力来划分，甚至回归终极目标，将同类项问题聚合并推导共性诉求。
4. **拆解目标，达成共识，策略介入**（落地）。从终极目标拆解成若干个子目标，并通过相应的策略机制等保证各方达成共识、目标落地。

### 配套：如何画因果回路图（CLD）
因果回路图（又称系统循环图）以因果关系链路的形式描述影响系统行为的结构。五个心得：
- **从哪里入手**：你最关心的问题是什么？系统想要达成的目标是什么？
- **如何定义变量**：系统的驱动力是什么？在与希望解决问题相关的因素中，哪一个最关键？
- **如何识别关键变量**：通过深度访谈/小范围团队讨论，就最重要的因素拉齐认知、达成共识。
- **何时定义连接关系**：随着思考脉络，确定链接类型（增强/调节/延迟）。
- **如何定义系统的边界**：通过共创会研讨，同时时刻提醒自己要研究的是什么。

## 结果解读
- 输出是一张/多张因果回路图与一套"调研-监测"体系框架：能看清目标、关键变量及变量间的回路关系（尤其是增强回路）。
- 警惕"表象陷阱"：看到的事件表象（用户描述）只是冰山一角（冰山理论），不一定是真正的问题，需透过背后的行为洞察更深层问题与根因。
- 案例·奶茶店的故事：小张从"进店人数变多"直接推断"大学生爱喝奶茶"并开店却生意惨淡。其错误在于——只关注"多"这一存量结果，忽视了"变"这一动态因素，"目标关注"设定错误、影响目标的"变量"过于局限。
  - 用 CLD 还原真相：小Q 是网红打卡店，学生消费后"分享转发"→ 网上知道的人越来越多 → 形成"知名口碑"→ 慕名打卡的人越来越多。网络知名度与消费人数组成一个**增强回路**，因而生意火爆。
- 用户访谈中需警惕自己变成用户"传声筒"——觉得用户说什么就是什么、把所有反馈都当真需求，最易跳入表象陷阱。

## 工具 / 实现
- 思维工具：系统思维（目标-变量-连接）、因果回路图（CLD/系统循环图）、冰山理论；可与 5W1H、六顶思考帽等思维方式配合。
- 信息获取与对齐手段：深度访谈、小范围团队讨论、共创会、田野观察、录屏日志。

## 局限与误用
- 系统是高度复杂并且动态变化的，**解决问题没有万能的方法**，仍需审慎地思考分析、抛弃固有范式。
- 易踩坑：把用户描述的表象当成真需求（"传声筒"/表象陷阱）；只关注单个变量而忽视连接与目标；只看存量结果（"多"）而忽视动态因素（"变"）。
