# 文本分析

> 来源：神灯圈子·焦玉霞《浅谈如何用文本分析助力用研》（原作者赵洋帆，2022.10.31） 原文 http://xingyun.jd.com/shendeng/article/detail/46467

> 一句话：把用户主动留下的非结构化文本（评价、反馈、社媒分享）通过采集、分词、关键词与情感挖掘结构化，快速获取针对研究问题的用户输入。
> 落点：`methods/toolbox/analysis/`。判据——这套手段**能脱离原研究被单独复用**（分词、TF-IDF 关键词、情感分析、语义网络可用于任何海量非结构化文本）。

在日常研究工作中，我们针对研究问题进行问卷或访谈，通过结构化或半结构化的方式获取信息再推出结论。但网络上已经存在大量用户主动分享的信息——用户评价、用户反馈、社交媒体分享。文本分析可以将这些原有的非结构化信息结构化，获取到针对研究问题的用户输入。

## 何时用

- 适合的问题类型（原文三类场景）：
  - **场景1**：问卷开放题回收了千百条用户反馈，逐条编码费时更废人。
  - **场景2**：业务临时提需想知道用户对某些问题的想法，但时间紧、难以快速完成调研输入反馈。
  - **场景3**：对于不熟悉的品类研究，想提前了解用户对这个品类都在关注什么。
- 可针对的研究场景很丰富：
  - **产品研究**：通过评价和社媒分享了解用户的产品关注点、痛点和爽点；结合时间分析可了解发展趋势变化；也可了解版本更迭的舆情情况。
  - **营销研究**：监测活动声量、活动舆情、用户画像。
- 与近似技术的区别：
  - 本篇偏**通用文本技术本体**（采集 → 分词 → 关键词/情感/语义网络的方法与工具）。
  - `methods/toolbox/analysis/voc-analysis.md`（VOC 用户之声）偏**来源与闭环**：强调多源汇聚、编码本、把洞察分发到责任团队并跟踪修复（close the loop）。两者技术同源，VOC 可视为文本分析在"持续监测 + 闭环"场景下的体系化落地。

## 输入数据要求

- 需要先有**文本数据**：巧妇难为无米之炊，文本分析第一步是获取文本。
- **数据类型不止"用户在说什么"**：针对一条用户评价反馈，除了爬取用户说了什么，还可同时爬取用户画像、评价打分、评价商品名称、评价时间、互动数据等。结合这些信息可进一步分析有价值的问题：
  - 不同画像的人关注点有什么差异；
  - 不同评价时间的评价有哪些区别；
  - 好评和差评的用户关注点分别是什么。
- 数据进来后**不能直接使用**，需经预处理（见下"分析步骤"第 2 步）。

## 分析步骤

文本分析的标准链路为：数据采集 → 数据预处理 → 文本分词 → 文本信息挖掘（关键词 / 情感 / 语义）。

1. **数据采集**：推荐**八爪鱼**，对新手友好，内置常用网站常用文本采集模板（如微博、天猫评价）；有特殊需求也可自定义采集模板。缺点是只能采集网页数据，**app 端的信息需要用 python 写爬虫实现**。
2. **数据预处理**：
   - **去噪**：例如分析某营销活动下真实用户的反馈时，需删除营销号/大 V 的推广信息。营销号往往有很多点赞/评论转发，可通过对点赞评论数排序，方便定位并删除。
   - **格式处理**：把数据处理成可分析状态。例如爬取特定商品评价人数，爬到的是文本化数据（"2万+"），需再处理成数字（20000）才能做后续结构化分析。
3. **文本分词**：文本处理的第一步是把收集回来的文本断句分词，后续才能基于词语做结构分析。
   - 英文分词比较简单，通过空格即可分割单词。
   - 中文的最小语义单位是词，需要将连续的字分割成不同单词的组成。中文分词基本上基于分词词典：不同单词连续一起出现的概率存在差异，**通过计算不同单词组合出现的最大概率分割方式就是中文分词的基本原理**（感兴趣可看吴军《数学之美》隐式马尔科夫链）。网上有成熟的中文分词词典，如**结巴分词**，可直接利用。
4. **文本信息挖掘**：分词后即可做信息挖掘，常用方式见下"结果解读"。

## 结果解读

分词后常用的三种信息挖掘方式：

- **关键词挖掘（TF-IDF）**：要知道一句话里用户讲的最核心内容。注意：**出现越多的词不一定是最核心话题**。设想一篇文章通篇介绍某个人，人名可能在每段都是高频词；但第一段讲他的家乡、第二段讲他的工作，在各自段落里"家乡""工作"虽不如人名高频，却更代表该段核心话题。因此不能仅凭词频高低判断关键词，而可用 **TF-IDF**——如果某个词或短语在一个文档（反馈）中出现多次，但在其他文档（反馈）中很少出现，就认为它具有很好的区分性，适合用来表征这个文档。
- **情感分析**：用户表达痛点和爽点时会带有情绪，使用的单词带有正面/负面情感。可通过**情感语料库**判断用户在一句话中所用词语的正向/负向情况，从而整体判断其情感是正向还是负向。若获取的数据本身就有评分，也可直接根据评分判断文本情况、直接分类处理。
- **语义分析**：词频只是单个词语的含义，词语需放到语境里才知道在表达什么。语义分析可涉及文本聚类、语义网络、主题抽取：
  - **文本聚类**：通过现有语料，按每个单位语料的单词构成与其它语料的距离做聚类，被判为一类的语句往往表达相近语义。**但该算法对短文本特别不友好。**
  - **主题抽取 / 语义网络**：核心是根据每个单位中单词一起出现的概率计算，判别哪些词语构成比较相近从而分组。例如"客服"这个词最常和"专业""高效"一起出现，说明用户对客服的专业高效比较关注。

## 可视化 / 出图

- **词云 / 词频**：用 KH Coder 可快速查看大家主要在讨论什么（词频）并制作词云。
- **语义网络图**：展示哪些词与哪些词更紧密（如"普通"与"区别"意义高相关）。
- **分组对比**：按好评/差评等分组变量，分别输出代表性词语、语义网络等（见下"工具/实现"中的 KH Coder 分组分析）。

## 工具 / 实现

- **新手友好：KH Coder**
  - 安装后可直接导入数据分析，**完全不需要写代码**；可切换中文说明界面，按说明书约 5 分钟即可上手。
  - 支持中文分词；支持词性筛选、录入停止词、录入特定词汇与排除特定词汇，使分词结果更合理。适合做分词、看词频、制作词云。
  - 进一步支持信息挖掘：可直接做**文本聚类、语义网络分析**——不仅了解哪些词出现更多，还知道这些词与哪些词更紧密。对某一意群，可点击相关词语**直接回到原本语境**找到用户具体在说什么（例：点击"普通"发现很多用户提到"感知与普通会员没有差异"，提示需提高关注度）。
  - 支持**分组分析**：若已对文本打标（如好评/负向），可按该评分变量分组，获得分组的代表性词语、语义网络等。

## 局限与误用

- **仅凭词频判断核心话题**：高频词不一定是关键词，应改用 TF-IDF 衡量区分性（见"结果解读"）。
- **文本聚类用于短文本**：聚类算法对短文本特别不友好，短文本宜改用主题抽取 / 语义网络。
- **未去噪直接分析**：营销号/大 V 推广信息会因高互动量污染结果，需先按点赞评论数排序剔除。
- **未做格式处理**：文本化的数字（如"2万+"）未转成数值无法做结构化分析。
- ⚠️ 待补充(缺源)：原文未系统展开样本量、语料覆盖度、情感语料库领域适配等更细的方法学边界。
