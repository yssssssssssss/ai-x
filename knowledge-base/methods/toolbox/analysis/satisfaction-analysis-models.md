# 满意度分析模型（四分图/Kano/AHP/SEM）

> 来源：神灯圈子·顾小妍《产品、系统、功能、服务的"万能温度计"——用户满意度》 原文 http://xingyun.jd.com/shendeng/article/detail/46493 （本文原作者徐凯，写于 2019.11.22）

> 一句话：用四类满意度模型完成满意度体系搭建或满意度数据分析——四分图诊断改善优先级、KANO 给需求分类排序、层次分析（AHP）定指标权重、结构方程（SEM）做理论假设检验并求权重。
> 落点：`methods/toolbox/analysis/`。判据——这四套手段都**能脱离单次满意度研究被单独复用**（如四分图/IPA 能用在满意度之外的大量属性评估，AHP 能用于任何多准则权重决策）。

用户体验已逐渐成为企业之间竞争的重要筹码，而用户满意度作为衡量用户体验的重要指标，在企业具体的用户体验实践中得到了较大范围的应用。在京东，满意度调查不仅应用于 APP 产品体验的评估，帮助 APP 各产品线周期性监控用户满意度变化、发现各产品线的体验优化点；而且应用于商家店铺运营各环节的满意度评估，评估京东与商家合作过程中各环节的系统、规则、服务体验；甚至在衡量京东大厦餐厅就餐体验时也发挥着重要的指导作用。满意度已然成为京东相关产品、系统、功能、服务体验的"温度计"——通过满意度调查可以追踪其"绩效"表现、直观了解用户或客户的体验状况、对比与竞品的满意度水平差距、评估满意度改进措施的实际效果。

在满意度的实际应用当中，常用以下**四种满意度模型**来帮助完成满意度体系的搭建或满意度数据的分析。

## 何时用

- 适合的问题类型：搭建满意度指标体系、分析满意度数据、定位需重点改善的指标、对需求/功能做分类与优先级排序、为各指标设定权重。
- 四模型分工速览：
  - **四分图模型**：诊断分析模型，靠"权重得分×满意度得分"二维四象限，回答"目前哪些指标需要重点改善"。
  - **KANO 模型**：分析用户需求对满意度的非线性影响，对需求/功能做分类与优先级排序；**并非满意度的计算模型**。
  - **层次分析模型（AHP）**：在已有满意度层级结构之上，构造判断矩阵来计算各层级指标的权重；**不负责告诉你如何构建层级结构**。
  - **结构方程模型（SEM）**：借助既有理论做假设检验的统计建模，构建并检验满意度模型、得到各指标权重。
- 与近似技术的区别：四分图与〈满意度驱动因素分析〉同源于 IPA 的"重要性×表现"四象限思路，但四分图横/纵轴用的是**权重得分与满意度得分**（权重可来自权重量表，亦可由 AHP/SEM 求得），驱动因素分析则用**派生重要性**（由属性评分与总体评分的统计关系反推）。两者可互为补充。

## 输入数据要求

- **四分图**：各指标的满意度量表数据 + 指标权重量表数据（用于算满意度得分与权重得分）。
- **KANO**：针对每个功能/需求的"正向 + 反向"成对问题的问卷数据（用于交叉表归类与 Better-Worse 系数）。
- **层次分析（AHP）**：已构建好的满意度层级结构 + 各层子指标两两对比的重要程度判断（构造判断矩阵）。
- **结构方程（SEM）**：依据理论/经验设定的满意度结构模型 + 对应观察变量的满意度数据。

## 分析步骤

### 1. 四分图模型

四分图模型，满意度的一种分析诊断模型，主要根据各指标的权重得分及满意度得分构建二维坐标，通过划分四象限来分析目前哪些指标需要重点改善。

![](https://img11.360buyimg.com/ling/jfs/t1/100927/20/2845/181621/5dd78386Ee1db1dc7/dd6a686487b28099.png)

在实际工作中应用四分图模型时需要重点完成以下几方面的工作：

- **根据研究对象确定反映用户满意度的指标。** 获取满意度指标可以参考现有的理论资料，也可以针对研究对象进行相应的用户调研，了解用户在与研究对象交互过程中所重点关注的内容有哪些，以此提炼出满意度指标。举个简单的例子，如用户对电商平台某类商品的满意度，可以分别从商品质量、商品价格、商品丰富度三个方面进行满意度评估。
- **根据满意度指标设计满意度量表和指标权重量表。** 关于量表的设计有比较系统的方法，在此不做赘述，需要注意的是在项目时间允许的情况下，对量表进行信度和效度检验是必要的。但是，在实际工作中，本着快速发现改善用户满意度切入点的目的，即使没有资源或时间来进行严格信效度检验，直接应用量表或问卷进行用户调查也是可以的。

![](https://img13.360buyimg.com/ling/jfs/t1/102399/30/2588/225281/5dd78416E0d9d053d/2ccf154d6ee9c75e.png)

- **满意度 & 权重数据收集及分析。** 满意度及权重量表数据收集之后，需要分别计算各指标的满意度得分及权重得分，得分的计算方式可以是算术平均，也可以是加权平均。根据各指标的满意度得分和权重得分，可将其绘制到由满意度和权重两个维度构成的坐标系中，再利用满意度得分和权重得分的平均值或中位值划分出四部分区域（修补区、机会区、优势区、维持区），结合各指标在坐标系中的分布，即可判断各个指标分别落入哪个区域中。

![](https://img30.360buyimg.com/ling/jfs/t1/103767/34/2836/184751/5dd78aa3E8ed4eb13/9335eebc45c3005f.png)

### 2. KANO 模型

KANO 模型，以分析用户需求对用户满意度的影响为基础，利用用户需求与用户满意度之间的非线性关系，对用户需求进行分类和优先级排序。

![](https://img13.360buyimg.com/ling/jfs/t1/59834/30/15875/244376/5dd792d2E18c863ef/6636fea6173daf1a.png)

KANO 模型并非满意度的计算模型，主要通过结构化的问卷和分析方法来对相关功能、需求进行分类和定位。可通过计算特定功能/需求在各个属性上的用户比例来判断功能/需求属于 KANO 模型中的哪种属性，也可通过 Better-Worse 系数进一步对功能/需求进行分析和优先级排序。

接下来以"京东店铺评分是否应该包含商品质量考核"为例（京东店铺评分是店铺商品质量、店铺售后服务质量、店铺物流速度等多个维度的综合指数，目前店铺评分已修改为店铺星级），简要说明 KANO 模型的具体应用方法：

- **设计问卷。** 针对"店铺评分包含商品质量考核"设计正反向问题，分别收集用户对店铺评分包含商品质量考核时的态度和不包含商品质量评分时的态度。

![](https://img10.360buyimg.com/ling/jfs/t1/74951/4/15748/201538/5dd794e6E257b264c/72cc73f7bf991c31.png)

- **问卷调查及数据分析。** 针对目标用户投放问卷并回收，对数据进行清洗后，数据分析时主要利用交叉表来统计不同选项上的用户比例。根据交叉表中各个单元格的属性定义来统计计算"商品质量考核"在各个属性上的用户选择比例。其中 S(A)、S(I)、S(O)、S(M)、S(Q)、S(R) 分别表示所调研的功能/需求在**魅力属性、无差异属性、期望属性、必备属性、可疑属性、反向属性**上的用户比例之和。从案例中可以得出，"商品质量考核"在 S(M) 上的用户比例最高，因此属于**必备属性**。由此可得：如果店铺评分包含"商品质量考核"，用户满意度并不会明显提升；但如果店铺评分不包含"商品质量考核"，用户满意度会急剧下降。
- **计算 Better-Worse 系数。** 为了进一步验证功能/需求的增加或消除对满意度的影响程度，可以分别计算其 Better-Worse 系数来分析。**Better 系数**可理解为增加功能/需求后的满意度系数，数值通常为正，代表如果提供某种功能，用户满意度会提升；正值越大或越接近 1，表示对用户满意度的影响越大、满意度提升越快。**Worse 系数**可理解为消除功能/需求后的不满意度系数，数值通常为负，代表如果不提供某种功能，用户满意度会降低；值越接近 -1，表示对用户不满意度的影响越大、满意度下降越快。在本例中，商品质量考核的 Better 系数为 0.40，Worse 系数为 0.63。由此可见，如果店铺评分不包含商品质量考核，会导致用户满意度快速下降。

![](https://img11.360buyimg.com/ling/jfs/t1/103246/13/2869/206175/5dd7956eEe806af4c/f4c53f520d8f09ad.png)

### 3. 层次分析模型（AHP）

层次分析模型，借鉴运筹学的层次权重决策分析方法，对满意度指标进行层级划分，并构造判断矩阵来计算各层级指标的权重，以此构建满意度指标体系。

![](https://img30.360buyimg.com/ling/jfs/t1/69005/3/15835/225282/5dd795e2E7b726f08/b34d67358efb28d4.png)

层次分析并不能告诉大家怎么来构建满意度的层级结构，层次分析的作用是在我们对满意度完成了层次结构的构建之后，帮助我们设定各层级指标元素的权重。满意度层次分析模型构建的主要工作有：

- **满意度层次结构构建。** 层级结构创建可参考现有的理论或资料，也可以通过探索性的研究或利益相关人的调研来梳理满意度的层级结构。

![](https://img12.360buyimg.com/ling/jfs/t1/53276/32/16371/216600/5dd7963dEe137a971/a1695b01f31eca87.png)

- **构造判断矩阵。** 判断矩阵是对某一指标 C 的子指标 B1、B2……Bn 两两对比得到的矩阵，其中 bij 表示对于指标 C 来说，子指标 Bi 与子指标 Bj 相比，其重要程度等级。Bij 的值是根据资料数据、专家意见或评价人经验确定的。另外，因判断矩阵为正互反矩阵（Bij>0，且 Bij=1/Bji），所以在实际构造判断矩阵的过程中，只需对比得出矩阵对角线上半部分的重要程度等级，然后对上半部分数据依此取倒数填写至矩阵下半部分对应位置即可。

![](https://img20.360buyimg.com/ling/jfs/t1/63528/22/15824/289528/5dd796a0E3ec5dafe/600ac5c88a54d4b8.png)

- **计算指标权重。** 指标权重的计算方法有两种：和积法和方根法。为方便理解，此处结合实际案例说明：假设京东支付的满意度包含四个指标——支付安全性、支付便捷性、支付方式多样性、支付页面美观性，判断矩阵及和积法、方根法的计算过程如下图所示。

![](https://img12.360buyimg.com/ling/jfs/t1/59315/31/16317/213219/5dd79708Ec6fcdbc0/7754b5f5092cc3d6.png)

![](https://img11.360buyimg.com/ling/jfs/t1/59708/19/15921/176462/5dd79a4bE7789def1/d17e194c4092cbf1.png)

![](https://img20.360buyimg.com/ling/jfs/t1/67997/9/16097/210722/5dd79742Ee71ad386/6e7d26de8ab7461a.png)

### 4. 结构方程模型（SEM）

结构方程模型，一种借助于既有理论进行假设检验的统计建模技术，在满意度的应用中需要通过现有的理论或经验构建满意度模型，并利用统计检验方法对模型进行检验，以此构建满意度模型，并得到满意度各指标的权重。

![](https://img13.360buyimg.com/ling/jfs/t1/57382/21/16376/214903/5dd79ba0E455627db/f62c55598be861ca.png)

结构方程模型在满意度研究中最典型的应用案例便是 CSI（顾客满意度指数模型）；在顾客满意度指数模型中，最有影响的三个模型分别为瑞典顾客满意度指数模型 SCSB、美国顾客满意度指数模型 ACSI、欧洲顾客满意度模型 ECSI，都是用来监测国家宏观经济情况的经济学模型，与 GDP、GNP 属于同一类型的宏观经济指标，而不是针对企业的诊断指导，不涉及企业产品/服务的具体绩效指标，因此在企业层面进行满意度调查时很少使用这些满意度指数模型。但可以借鉴结构方程的高阶验证性因素分析来进行指标权重的计算。

使用 AMOS 软件进行满意度权重分析的主要步骤可以分为：

- **绘制结构模型**，依据已有的经验或理论绘制满意度结构模型，在下图中绘制的是购物体验的满意度验证性因素分析模型。

![](https://img11.360buyimg.com/ling/jfs/t1/92603/24/2905/229843/5dd79bf3E1c6967ac/1d1d707afa96907d.png)

- **选取数据和变量**，在 AMOS 中打开对应的满意度数据，并将数据文件中的变量对应到上一步绘制的模型中的观察变量上，建立对应关系。

![](https://img14.360buyimg.com/ling/jfs/t1/65413/28/15980/110101/5dd79d6dE4dbb7fa4/08878e201ff7d7a3.png)

- 使用 AMOS 进行数据计算分析，得出相应的模型拟合结果以及标准化结构系数，从而得到相应的权重。

![](https://img30.360buyimg.com/ling/jfs/t1/59846/37/16083/245815/5dd79db8E8e96feb7/d33c8d4bcd3f58cd.png)

## 结果解读

- **四分图**：各指标落入修补区 / 机会区 / 优势区 / 维持区四象限之一，结合分布判断哪些指标需重点改善。
- **KANO**：得到功能/需求所属属性（魅力 A / 期望 O / 必备 M / 无差异 I / 可疑 Q / 反向 R）；Better 系数（接近 1 → 提供该功能满意度提升越快）与 Worse 系数（接近 -1 → 不提供该功能满意度下降越快）联合判断增删该功能对满意度的影响。
- **层次分析（AHP）**：得到各层级指标权重；须通过一致性检验（CR<0.1）方可采信，否则返回调整判断矩阵。
- **结构方程（SEM）**：得到模型拟合结果与标准化结构系数，由系数得到各指标权重。

## 可视化 / 出图

- 四分图：满意度（一维）× 权重（另一维）的二维四象限坐标图。
- KANO：正反向问题交叉表（含各单元格属性定义）；Better-Worse 系数表/散点。
- 层次分析：层级结构图、判断矩阵、权重与一致性检验计算表。
- 结构方程：结构模型路径图（验证性因素分析模型）及标准化结构系数。

## 工具 / 实现

- **层次分析（AHP）**：和积法 / 方根法手算可行；计算量大时可借助层次分析计算软件辅助权重计算与一致性检验。
- **结构方程（SEM）**：主要工具有 **LISREL、AMOS、Mplus**——其中 LISREL 和 Mplus 以语法为主，AMOS 提供图形化操作界面。

## 局限与误用

- 满意度模型只是一种用户体验的监测工具、一种了解用户感受的方法。对于实际的产品、系统、功能、服务来讲，即使不使用任何模型，单纯收集用户的满意度反馈，也能提供有效的用户体验信息。拥有"以客户/用户为中心"的用户体验先行意识，是有效应用各种研究方法/模型的重要前提。
- **KANO 不是计算模型**：它只做需求分类/定位，不直接产出满意度数值。
- **层次分析不构建结构**：AHP 只在已有层级结构上算权重，不负责梳理层级；且判断矩阵需通过一致性检验，否则结论不可用。
- **CSI 类指数模型慎用于企业诊断**：SCSB/ACSI/ECSI 属宏观经济指标，不针对企业具体绩效，企业层面满意度调查很少直接使用，仅可借鉴其高阶验证性因素分析思路算权重。
