# 购物篮分析（关联分析）

> 来源：神灯圈子·吴佳毅《购物篮分析：如何通过关联分析，让跨品类营销选品更高效》 原文 http://xingyun.jd.com/shendeng/article/detail/46744

> 一句话：从大量用户订单中挖掘"哪些商品经常被一起购买"的关联规则，为跨品类联合营销选品、商品推荐与捆绑促销提供数据依据。
> 落点：`methods/toolbox/analysis/`。判据——这套手段能脱离原研究被单独复用（电商推荐、捆绑促销、保险欺诈识别、资讯个性化推荐等场景通用）。

## 何时用
- 适合的问题类型：
  - 跨品类联合营销选品——给定一个购买行为（如买了手机），找出真正高概率连带购买的品类（而非凭直觉圈人，避免"发1万张券仅2张转化"式的人群不精准）。
  - 商品推荐 / 捆绑促销活动设计——找出"购买 A 后大概率购买 B"的组合。
  - 异常组合识别——如保险业中不常见的索赔要求组合可能提示欺诈，需进一步核实。
  - 内容个性化——如资讯业按兴趣匹配度推荐内容。
- 与近似技术的区别：关联分析关注的是"事件一起发生"的共现规律（得到可信且有代表性的规则），不依赖预先设定的因果假设。

## 输入数据要求
- 需要用户**订单 / 交易明细**数据：每笔订单包含用户购买的商品清单。
- 量纲：分析前需将底层的**订单表**（用户购买的商品明细）转化为**事务表**（用户是否购买清单中的某商品），即把数据转化为易于计数的**布尔型变量**，便于寻找频繁项集。
- 样本量：实际应用中订单量大、变量多，需要借助建模或编程工具完成计算。⚠️待补充(缺源)：原文未给出最小样本量门槛

## 分析步骤

整体思路（Apriori 算法）：通过"连接"和"剪枝"找到**频繁项集**，再由**置信度**筛选获得**关联规则**。

**关键概念**

- **频繁项集：** 经常出现在一起的事件的集合。
- **支持度（Support）：** 两件商品（A∩B）在总销售笔数（N）中出现的概率，即 A 与 B 同时被购买的概率。
- **置信度（Confidence）：** 购买 A 后再购买 B 的条件概率 P(B|A)。比例越大，说明购买 A 的客户很大期望会购买 B。
- **提升度（Lift）：** 衡量按规则推荐相对随机推荐的效率提升，$lift(a \rightarrow b) = P(b|a) / P(b)$。

![](https://img20.360buyimg.com/ling/jfs/t1/113103/24/11464/122637/5efdaed5Ec7827b48/759207dabaf224f6.png)

**第一步：找到所有满足最小支持度的频繁项集**（示例最小支持度为 0.2）

1. 扫描订单数据，得到每个候选项的支持度，与最小支持度比较，得到 1 项频繁项集 L1。
2. 将 L1 与 L1 连接得到候选项 C2，计算每个候选项的支持度，与最小支持度比较，得到 2 项频繁项集 L2。
3. 将 L2 与 L1 连接得到候选项 C3；连接过程中可用 **Apriori 原理**（一个频繁项集的任一子集也是频繁项集）剪枝，减少计算量；比较 C3 的支持度与最小支持度，得到 3 项频繁项集 L3。
4. L3 与 L1 连接，根据 Apriori 原理未找到符合条件的 4 项频繁项集，第一步完成。

![](https://img13.360buyimg.com/ling/jfs/t1/140093/6/1901/81946/5efdaef3Eba37f839/05b7f80661ba1a21.png)

**第二步：通过置信度筛选符合条件的关联规则**

1. 根据频繁项集，得到关联规则的前项和后项，计算每条规则下的支持度与置信度。
2. 根据对置信度的要求，筛选得到所有符合条件的规则。
3. 示例中经两步计算，得出满足"支持度≥0.2、置信度≥0.6"的 16 条关联规则。制定推荐策略时，可根据对推荐精度的要求选择适合的规则。

## 结果解读

输出为一组**关联规则**（前项 → 后项），每条带支持度、置信度、提升度等指标。读法以一条规则为例（"规则标识 3"，前项＝冻肉∩灌装蔬菜，后项＝啤酒）：

- 历史数据中同时购买"冻肉"和"灌装蔬菜"的顾客有 173 人，占总人数 **18.424%**（前项支持度）。
- 这些顾客中有 **84.393%** 也同时购买了"啤酒"（置信度 P(啤酒 | 冻肉∩灌装蔬菜)）。
- 同时购买三种商品的顾客占比 **15.548%**（规则整体支持度）。
- 采用此规则向顾客推荐啤酒，比随机推荐提升效率 **2.705 倍**（提升度 lift）。
- 历史库中买了冻肉和灌装蔬菜但没买啤酒的顾客占总人数 **2.875%**，即约 2.875% 的顾客可按该规则部署销售策略（可触达增量）。

结论方向：选取支持度、置信度、提升度均高的规则作为跨品类选品 / 推荐依据——既保证组合够常见（支持度）、连带购买概率够高（置信度），又确实优于随机（提升度 > 1）。

## 局限与误用
- 关联规则只反映**共现关系**，不等于因果——高置信度的"A→B"未必意味着推 A 就能促成 B，仍需结合优惠力度等其他刺激因素（原文案例中，券核销低除人群不精准外，另一原因即"优惠券力度刺激弱"）。
- 仅靠置信度高易被高频商品误导，应同时看**提升度**（lift > 1 才说明优于随机推荐）。
- 阈值（最小支持度 / 置信度）设置影响规则数量与精度：过松规则泛滥、过严漏掉长尾机会，需按推荐精度要求权衡。
