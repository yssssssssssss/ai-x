# VOC 产品线/模块分类法（京东 App 体验反馈）

> 来源：内部已训练的「VOC 自动分类 Agent」提示词（电商 / O2O 产品体验分类），转录留档为可复用正典。
>
> 一句话：把非结构化、口语化的用户反馈，按**封闭枚举**的京东 App 模块词库精确归类为 `level1(产品/服务) × level2(B2C/O2O) × level3(模块)`，再叠加**体验维度**与**优先级(A/B/C)**，产出结构化、防幻觉、可落地的问题描述。
> 落点：`methods/toolbox/analysis/`。判据——这套**模块词库 + 路由规则 + 维度/优先级口径**能脱离任一具体反馈被单独复用，是 [[voc-analysis]] 第 4-5 步「建立标签体系 + 打标」在京东 App 场景下的**具体落地实例**。

本篇是 [[voc-analysis]]（通用 VOC 流水线）的**京东场景特化**：voc-analysis 给的是「清洗→编码→聚类→情感→趋势」的通用链路与通用标签模板（模块×问题类型×情感×严重度）；本篇给的是京东 App 反馈打标时，`模块` 维度该用哪套**封闭词库**、`问题类型` 该用哪 11 个**体验维度**、`优先级` 怎么用**范围×严重度**算，以及一套**防幻觉 / 防劫持**的路由规则。目标是让多人/多批打标口径一致、模块不被臆造、跨界场景正确归属——直接提升编码质量。

## 何时用

- **适合**：京东 App / 小程序的用户反馈（应用商店评价、客服工单、问卷开放题、社媒吐槽、应用内反馈），需要按**京东产品线/模块**精确归类，并给出体验维度与修复优先级时。
- **与 [[voc-analysis]] 的关系**：voc-analysis 是主流程与通用模板；本篇是其中「标签体系」的京东实例。**先按 voc-analysis 跑流水线，遇到 `模块` 维度就用本篇的封闭词库与路由规则。**
- **不适合**：非京东产品的反馈（词库不匹配，应回退到 voc-analysis 的通用模块维度）；需要做显著性检验 / 平台优劣势对比（那是 `survey-crosstab-analysis`）；访谈逐字稿深读（那是 `qualitative-coding`）。

## 输入数据要求

- 单条用户反馈文本；若含**截图 / 图片 URL**，需先 OCR 提取页面文字、Tab、按钮、弹窗等视觉上下文——本分类**强依赖视觉上下文**判定模块，严禁在有图时盲猜。
- 反馈可能是口语化吐槽、纯情绪、或结构化工单（`【模块】【编号】【反馈】`）。
- 无最小样本要求；逐条独立分类。

## 输出结构（JSON）

每条反馈输出一个 JSON，**必须先输出 `analysis` 展示推导链，再按"先定 level3 再逆推 level1/level2"的顺序填字段**：

```json
{
  "analysis": "0.插件/OCR状态 1.视觉OCR要点 2.文本特征 3.字典红线核对 4.溯源逆推(先定level3) 5.白名单核验(level3是否一字不差在词库中,否则强制改兜底)",
  "level3": "从模块词库精确提取的三级模块名（最先确定）",
  "level1": "产品 / 服务 / 无效（由 level3 的词库烙印逆推）",
  "level2": "B2C / O2O / 无效（由 level3 的词库烙印逆推）",
  "dimension": "11 个体验维度之一",
  "priority": "A / B / C",
  "issue": "精简客观重述字面事实；原因未知就直言未知，严禁'可能由于…导致'式推测"
}
```

## 分类步骤

1. **清洗熔断**：空 / 纯空格 → 熔断判 `无效`。纯发泄、极短无意义（<4 字）→ `无效`。**意图豁免红线**：只要含明确业务/资产名词（如"打折券""红包"），无视字数与情绪，强制穿透提炼诉求，不熔断。
2. **（有图先 OCR）**：提取页面 Tab、按钮、弹窗、红底高亮坑位等，作为最高权重线索。
3. **先定 level3 模块**：从下方**封闭词库**精确提取（一字不差）；用「防劫持路由速查」处理跨界 / 易混场景。
4. **逆推 level1 / level2**：每个模块带「词库烙印」（见下），据此反推 `产品/服务` 与 `B2C/O2O`。
5. **白名单核验（反幻觉）**：复核 level3 是否**一字不差**存在于词库；若为臆造词（如"分享""下载""厨具页面""外卖秒杀"）或把分类标题（如"用户支撑""营销活动"）误当标签 → 强制改兜底 `其他`(B2C) / `外卖其他`(O2O)。
6. **定 dimension**：从 11 个体验维度精确提取（见 C 部分）。
7. **定 priority**：按「范围 × 严重度」打分映射 A/B/C（见优先级一节），注意降级红线。
8. **写 issue**：客观重述字面现象，去情绪黑话，**严禁过度推理 / 脑补归因**。

## A. 模块词库（level3 封闭枚举 · 必须一字不差）

> **绝对白名单封闭集合**：严禁臆造任何不存在的标签；严禁把加粗的**分类标题**（如"交易相关""用户支撑"）当作标签；边缘场景找不到对应 → B2C 强制兜底 `其他`，O2O 强制兜底 `外卖其他`。

### 1. 服务体验类（词库烙印：选此类则 `level1=服务`）
`外卖投诉` · `商品质量` · `物流履约` · `客户服务` · `金融客服`

- **外卖投诉**（level2=O2O）：O2O 履约问题（奶茶/送错/超时/凉了/骑手/门店/吃坏肚子/餐具缺失）。**绝对互斥红线**：`level1=服务 且 level2=O2O` 时，level3 **仅能是** `外卖投诉`；若诉求是外卖退款/写评价等 **App 功能操作**，须把 level1 改回 `产品`。
- **商品质量**（level2=B2C）：实物质量差 / 假货 / 瑕疵（未提 App 操作）。
- **物流履约**（level2=B2C）：B2C 快递慢 / 不送货上门 / 丢件 / 暴力分拣（非外卖）。
- **客户服务**（level2=B2C）：找人工、投诉态度、客服不理人、异常订单咨询、**账号注销/合并与数据同步**等需后台人工介入的通用求助。
- **金融客服**（level2=B2C）：【高危易错】仅限明确指向**底层金融工具**（银行卡/信用卡/金条/白条/钱包余额）的单点人工核查。**禁区**：异常订单、订单被拦截/冻结、普通退款没到等**履约层**异常一律归 `客户服务` 或对应产品模块，严禁归此。

### 2. 产品体验 · O2O 本地生活（22 个 · 词库烙印：`level1=产品, level2=O2O`）
核心 21：`秒送频道` `秒送搜索` `秒送商品详情` `秒送购物车` `秒送结算页` `秒送店铺页` `秒送订单中心` `秒送售后/评价/赔付` `外卖频道` `外卖搜索` `外卖店铺页` `外卖商品详情` `外卖购物车` `外卖结算页` `外卖订单中心` `外卖售后/评价` `酒旅` `饭卡` `到店团购` `家政` `京东到家`
兜底 1：`外卖其他`

> O2O 强制雷达：原话或 OCR 含「秒送 / 外卖 / 起送 / 起步价 / 配送费 / 门店 / 自提 / [自营]秒送」→ 优先进 O2O 词库，严禁默认回落 B2C。**视觉绝对优先例外**：顶部 Tab 高亮选中的是"首页"等全局通用 Tab 时，无视下方秒送/外卖字眼，强制 B2C。

### 3. 产品体验 · B2C 电商（62 个 + 兜底"其他" · 词库烙印：`level1=产品, level2=B2C`）
- **交易相关(18)**：`首页` `商品详情页` `店铺` `凑单页` `购物车` `结算页` `收银台` `支付成功页` `订单中心` `售后页面` `评价` `问答` `视频` `直播` `排行榜` `点评` `价保` `发票中心`
- **搜索推荐(3)**：`搜索` `推荐` `扫啊扫/拍照购`
- **用户支撑(15)**：`登录` `注册` `个人中心` `设置` `地址管理` `账户安全` `浏览记录` `商品收藏` `店铺关注` `消息` `短信` `钱包` `在线咨询` `余额` `小金库`
- **营销活动(13)**：`优惠券` `红包` `礼品卡` `京豆` `秒杀` `百亿补贴` `会员中心` `Plus会员` `省钱卡` `京东校园` `活动会场` `东东农场` `种豆得豆`
- **其他业务(13)**：`白条` `以旧换新` `充值中心` `试用` `看病买药` `京东超市` `拍卖` `9块9频道` `领券中心` `夺宝岛` `新品` `京言` `特价频道`
- **兜底(第63个)**：`其他`——无法归类的 App 体验问题（启动慢、字体小等）。

## B. 防劫持 / 跨界路由速查（打标防呆红线）

> 语法：[条件特征] → [强制归属]（禁 [易误判模块]）。高频易错，打标前必查。

- **O2O 漏斗与打包费最高主权**：保洁/维修→`家政`；车票→`酒旅`；文本/OCR 含"打包费"或"必选不合理"——**无论是否同时报"下不了单"**，最高权重锁死 `外卖店铺页`（斩断结算页的动作劫持）；仅纯系统报错致无法下单且无打包费槽点→才归 `外卖结算页`；"秒送"骑手异常→`秒送订单/店铺`；点不了 + OCR"无法出餐"→`[服务-O2O-外卖投诉]`。
- **系统与跳转**：开屏/摇一摇广告→`首页`(禁消息)；锁屏/通知栏广告→`消息`；赚钱关不掉/跳逛播视频→`视频`；领券/补贴前置弹窗→`优惠券`；跳过选品直达付款→`结算页`；下单后对公转账→`支付成功页`。
- **容器 UI 防呆**：底 Tab"我的"+话题遮挡→`个人中心`；购物车切 Tab 残 O2O 单/含清空勾选→`购物车`(外卖浮层禁独立判)；搜本店遮通知栏→`店铺`；搜索框混敏感词→`搜索`；欢迎注册→`注册`；待评空白→`评价`；IM 聊天窗(深色/半屏/拒收失败)→`在线咨询`(禁消息/订单)。`购物车`必含清空/勾选/加减号；`结算页`必含地址及去支付(无地址绝非结算页)。
- **频道与视觉坑位**：OCR 红底/高亮选中"9.9包邮/9.9"→`9块9频道`；首页秒杀/直播卡片→对应 `秒杀/直播`(禁推荐)；顶部秒送外卖 Tab + 底流错乱→`秒送/外卖频道`；潮流好物→`推荐`。**频道/子 Tab 的红底白字高亮选中项视觉主权最高**，无条件高于外层父 Tab 与通用流 Tab。
- **黑话与内容**：逛 Tab→`视频`；"我京"+有效痛点(如"不要总改动/求自定义/求简洁")→兜底 `个人中心`(纯"非常好"等无意义词必熔断)；提及"创作中心"→`点评`(禁基础评价)。
- **会员公私域**：店铺卡/山姆/沃尔玛→`店铺`；Plus/付费/扣费/积分兑换/洗车/联名视频→`Plus会员`(禁家政/视频/会员中心)；学生认证→`京东校园`；仅提金银钻/成长值→`会员中心`(禁含积分兑换/扣费)。
- **资产与充值**：超市卡(买/绑/用报错)→`京东超市`(禁礼品卡/钱包/结算)；国补/家电补贴→`以旧换新`(禁优惠券)；发票+有奖/抽奖→`发票中心`；领货/提货码/卡密兑换→`礼品卡`(禁订单/物流)；水滴(非物理)→`东东农场`。
- **金融与提现（钱包强穿透）**：底层账户/解绑银行卡/不明扣费查账单→`金融客服`(禁钱包)；视频赚钱/逛播提现→`视频`；提现至微信/银行卡→`余额`；红包提现→`红包`；小金库提现→`小金库`；白条还款→`白条`；充话费没到→`充值中心`。钱包全局白屏/排版全乱(无具体资产/解绑操作)→才兜底 `钱包`。
- **生命周期时序**：途中改地址/发货前取消/买后分享→`订单中心`(禁商详/地址)；下单前新建/外卖点餐遇地址问题/定位选不到→`地址管理`(绝对禁被外卖结算页劫持)；顺手买缺失/跨页改数量→`结算页`(禁购物车)。
- **售后与服务隔离**：模糊要求退换/取消/退款遇阻且**无 App 操作痛点**→B2C 归 `[服务-客户服务]`、O2O 归 `[服务-外卖投诉]`；**明确指出退款页/退换按钮等 App 故障**→才归 `售后页面` / `秒送售后/评价/赔付`。
- **准入与实体隔离**：必须登录才能搜/看→`登录`(禁搜索/商详)；纯吐槽实物价高/乱涨价/标价错且无 App 断点→`[服务-商品质量]`。
- **优惠券跨界**：券 + 特定频道(9.9/超市/充值)→对应频道(禁大盘优惠券)；单张券无法用/过期/没显示需排查→`[服务-客户服务]`(特定资产核查属人工范畴)。
- **合规与未成年**：缺"未成年模式"入口→`设置`；因"保护未成年人"限看/无法购买(识别误伤)→`[产品-B2C-其他]`(禁商详/结算/普通风控)。
- **账号人工服务**：注销账号/合并账号/跨账号同步历史订单→`[服务-客户服务]`(禁个人中心/设置/账户安全)。

## C. 体验维度（dimension · 11 个封闭枚举）

> **反幻觉红线**：`dimension` 只能从以下 11 个精确提取；严禁输出"信息展示/操作流程/功能定义"等内部评分词。

1. **风控限制**：业务/系统主动触发的限流拦截，话术含"火爆/拥挤/太热情/财产安全"。隐性特征：简单提"无法领券/没有券/券和别人不一样/特价券入口消失"且无报错 → 多为账号命中防刷被静默隔离，强制 `风控限制`，严禁判功能缺失/视觉。结算报"销售火爆请关注其他活动"→ 同。
2. **系统异常**：明确技术故障（闪退/白屏/死机/乱码/报错码/无限转圈/结算价格异常计算）。
3. **业务规则**：对非 Bug、非风控的既定规则不满（入口关闭/资产不可叠加/门槛不合理）。动态价差（商详/购物车与结算页差价、国补预估差）强制归此，禁判配置错误/系统异常。
4. **配置错误**：静态人工录入失误（活动日期错/客服空号/文案错别字/错链死链/店铺资源位失误/违背物理常识的参数如超重 100kg）。
5. **功能缺失**：无法满足需求或流程断点。信息缺失直接阻断线下履约（如团购无适用门店）→ 升级为 `功能缺失`，禁当视觉瑕疵。
6. **算法体验**：搜索结果不匹配/不精准、推荐单一/重复、负反馈(不感兴趣)无效。
7. **易用性差**：层级深、操作繁琐、找不到入口、交互反人类——功能可用但难用。
8. **视觉/内容**：字小、排版乱、图加载失败、信息歧义（**仅限不阻断主流程的纯 UI 展示瑕疵**）。
9. **新功能建议**：用户明确提出的新需求/期望（Feature Request）。
10. **服务质量**：实体商品质量、商家/客服态度、物流速度（服务体验大类）。
11. **客服咨询**：针对单个具体问题的求助（"帮我查物流""怎么退款"）。

## D. 优先级（priority · 范围 × 严重度 → A/B/C）

- **范围**：[3] 核心链路/大面积(≥20%)；[2] 分支/特定机型(5-20%)；[1] 长尾/单店/个案(<5%)。
- **严重度**（仅用于算分，**不可作 dimension**）：[3] 核心阻断；[2] 部分阻断/困惑；[1] 轻微干扰。
- **映射**：得分 6 或 9 → `A`；3 或 4 → `B`；1 或 2 → `C`。
- **降级红线**：纯 UI 瑕疵(严重度=1)；搜索算法偏差、非阻断微小价差、非崩溃卡顿掉帧(严重度≤2，严禁 A)；局部死锁(如限购1/起购2，范围1严重度2 → 强制 `C`)；敏感内容推荐(严重度2范围1-2 → `B` 或 `C`)。

## 结果解读

- **输出形态**：每条反馈一个结构化 JSON（level1/level2/level3/dimension/priority/issue + analysis 推导链）。可批量汇总为「模块 × 维度 × 优先级」的频次表，直接接入 [[voc-analysis]] 的主题表与 Top 痛点清单。
- **怎么读**：level3 模块定位"问题出在哪个产品线/页面"；dimension 定位"是哪类体验问题"；priority 定位"先修哪个"；issue 提供可读的客观描述。
- **下结论**：模块 × 维度的高频高优(A) 组合即核心痛点；配合 issue 原话佐证，交对应模块的责任团队闭环。

## 多模态规则（有截图时）

1. **Z 轴前景优先**：居中操作阻断弹窗权重最高；纯广告穿透弹窗按背景归。
2. **多图末端优先**：多步截图以最后一步"案发现场"为准。
3. **目标模块优先**：已进入目标页即获主权，严禁被引导文字劫持。
4. **频道/子 Tab 最高视觉主权**：红底白字/红色气泡/高亮选中项 > 外层父 Tab > 通用流 Tab。
5. **UI 防呆**：`购物车` 必含清空/勾选/加减号；`结算页` 必含地址及去支付。

## 结构化工单降噪

遇 `【模块】【编号】【反馈】`（如【我京】【4】【太花哨】）：忽略中间数字编号；当【反馈】为纯情绪/无指导(如"非常好/没有")→ **无视前置标签强制熔断判 `无效`**；当【反馈】含具体痛点但缺页面名词→ 允许用前置【模块】(如"我京")做空间主权兜底，映射至对应业务(如 `个人中心`)。

## 局限与误用

- **词库会过时**：京东模块/频道随业务调整，词库需定期复核更新；用前确认是否仍是最新封闭集合。
- **强依赖视觉**：有截图却不 OCR 直接猜模块，极易误判（尤其频道主权、容器归属）。
- **过度推理**：脑补"可能由于…导致"的归因会污染 issue 与后续统计——只重述客观现象。
- **臆造标签 / 误用标题**：发散造词或拿分类标题当 level3，会破坏封闭枚举的统计口径——白名单核验是硬红线。
- **范围特化**：仅适用于京东 App 反馈；套到非京东产品会强行错配，应回退 [[voc-analysis]] 通用模块维度。
- **draft 状态**：本词库与路由规则转录自内部提示词、尚未在 wiki 评审；结论性使用前以最新业务事实复核。
