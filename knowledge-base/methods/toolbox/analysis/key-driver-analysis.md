# 满意度驱动因素分析（Key Driver Analysis）

> 来源：Joyspace《满意度驱动因素分析（Key Driver Analysis / 关键驱动分析）》 https://joyspace.jd.com/pages/VZCZVZmTocEPj84aV1qc

> 一句话：通过统计手段找出"哪些体验属性最影响总体满意度/NPS"，把有限的迭代资源投到真正撬动用户感受的少数关键属性上。

## 何时用

- **适合的问题类型**：迭代后做满意度/体验评估时，需要回答"下一步该先改哪里"；属性较多、资源有限、需排序取舍。本质上是以总体满意度（或 NPS、推荐意愿、留存意向）为因变量，各体验属性评分为自变量，量化每个属性对总体感受的"派生重要性"，并据此排定改进优先级。
- **在产品迭代期的定位**：本方法属于【效果评估 → 诊断归因】的桥梁——满意度问卷量出"有多好"，驱动分析回答"接下来先改哪、改了最划算"。最佳时机是一轮迭代上线、满意度/NPS 数据回收后，既复盘本轮，又为下一轮排期；它把模糊的"用户不满意"切成可执行的优先级清单，是迭代节奏中"评估→决策"的关键一环。
- **与近似技术的区别**：核心是区分两类"重要性"——用户口头说重要的（**陈述重要性**）≠ 真正驱动其满意度的（**派生重要性**）。后者由属性评分与总体评分之间的统计关系推导，再叠加"表现"维度，落到优先级矩阵。这与直接询问重要性的方法（陈述重要性）、以及只解释"为什么满意/不满意"的归因方法（如 ECT）有本质分工。

## 输入数据要求

- 一份含"总体满意度/NPS 单题 + 多个属性满意度单题"的问卷数据，通常为 5/7/10 点量表。
- **因变量**：总体指标（总体满意度 / NPS / 推荐意愿 / 留存意向），单题量表采集。
- **自变量**：基于用户旅程拆出的各体验属性满意度评分（如加载速度、易用性、客服、价格感知），每项一道满意度题，要求覆盖全、尽量互斥。
- 数据需经清洗（去无效答卷、处理缺失，见〈满意度问卷设计与数据清洗〉）。样本量不足、量表区分度差或天花板效应都会让重要性估计不可靠。

## 分析步骤

**陈述重要性 vs 派生重要性。** 直接问用户"这项有多重要"得到的是陈述重要性（stated importance）；它易受社会期许影响，且用户往往把每项都打成"重要"，区分度低。派生重要性（derived importance）则不问重要性，只采集"各属性满意度"与"总体满意度"，用统计关系反推哪项真正驱动总体感受。两者常不一致——这种背离本身就是洞察（如"客服"被嘴上说不重要，但其评分与总体满意度高度相关）。

**从相关到回归再到相对权重。** 派生重要性有三层做法，复杂度与稳健性递增：

- **相关分析**：算每个属性与总体满意度的相关系数。最直观，但每项单独看，忽略属性间叠加。
- **多元线性回归**：以总体满意度为因变量、所有属性为自变量，用标准化回归系数（standardized beta）衡量重要性。MeasuringU 的案例中"易用性 beta=.55、稳定性 beta=.12"即典型读法，R² 反映模型整体解释力（如 .50 表示自变量解释了一半方差）。问题在于：当属性高度相关（多重共线性），beta 会变得不稳甚至变号、跨样本漂移。
- **相对权重 / Shapley value**：专为相关自变量设计。Johnson（2000）的相对权重法（relative weights）先把相关自变量转换成一组彼此正交（不相关）且最大程度逼近原变量的"替身"，对替身回归后再把解释力映射回原属性，从而把 R² 无负值地分配给各属性；Shapley value 回归则源自博弈论，对所有变量进入顺序求平均贡献，结果跨样本更稳。R 语言 `relaimpo` 包（Grömping）的 LMG 指标即同类思路——对所有进入次序平均分摊方差。多数商业方案（Displayr 等）默认推荐 Johnson 相对权重或 Shapley，正是因为它们能给出加总为 100% 的稳健重要性。

**IPA 四象限。** 单有重要性还不够，需叠加"当前表现"。把横轴设为表现（属性满意度均值），纵轴设为重要性（派生权重），用均值划十字，得到重要性-表现分析（Importance-Performance Analysis, IPA, 源自 Martilla & James 1977）：高重要-低表现=**重点改进**，高重要-高表现=**保持优势**，低重要-低表现=**次要/低优先**，低重要-高表现=**锦上添花（过度投入）**。结论一目了然：**派生重要性高、表现低的属性，就是迭代优先项。**

**完整操作步骤：**

1. **定因变量**：明确总体指标（总体满意度 / NPS / 推荐意愿），单题量表采集。
2. **定属性清单**：基于用户旅程拆出体验属性（如加载速度、易用性、客服、价格感知），要求覆盖全、尽量互斥，每项一道满意度题。
3. **采集数据**：问卷投放，回收后做数据清洗（去无效答卷、处理缺失，见〈满意度问卷设计与数据清洗〉）。
4. **算派生重要性**：先看相关做体检，再用相对权重 / Shapley（或回归）算稳健权重，归一到 100%。
5. **算表现**：取各属性满意度均值或 Top-box 比例。
6. **画 IPA 四象限**：重要性×表现交叉，均值划线分区。
7. **出优先级**：锁定"高重要-低表现"为重点改进项，结合 Kano 与业务成本复核后落到迭代需求池。

## 结果解读

输出为各属性的相对重要性权重（常归一到 100% 或 R²）、重要性×表现四象限图、以及优先改进清单。读法的核心是把"重要性"与"表现"两维交叉，落到 IPA 四象限：

| 象限（重要性×表现） | 含义 | 行动 |
| --- | --- | --- |
| 高重要 · 低表现 | 痛点、最拖后腿 | **重点改进**，优先排期 |
| 高重要 · 高表现 | 核心优势 | **保持优势**，守住别退化 |
| 低重要 · 低表现 | 影响小 | **次要**，暂缓 |
| 低重要 · 高表现 | 投入过度 | **锦上添花**，可抽资源 |

**相对权重计算说明（直觉版）**：当属性彼此相关，直接看回归 beta 会"互相抢功"导致失真。Johnson 相对权重的做法是——把相关属性先正交化成不相关的替身、对替身做回归（此时贡献干净可加），再按替身与原属性的关系把解释力分回原属性，最终各属性权重非负且加总等于模型 R²，便于归一成百分比直接比较。

**结合 Kano 分层**：把属性按 Kano 分为基本型 / 期望型 / 兴奋型。基本型（must-be）做好了用户无感、缺失则强烈不满，故它在驱动分析里"重要性"可能被低估，但绝不能掉；兴奋型（attractive）有则惊喜、无则不怨，是差异化弹药。先用 Kano 给属性定性，再用驱动分析定量排序，避免"只盯回归数字而砍掉基本盘"。

## 可视化 / 出图

- **IPA 重要性-表现四象限图**：横轴为表现（属性满意度均值），纵轴为重要性（派生权重），用均值划十字分区，直接读出"重点改进 / 保持优势 / 次要 / 锦上添花"四类属性。
- **相对重要性权重条形图**：把归一到 100% 的各属性权重排序展示，一眼看出谁最撬动总体满意度。

## 工具 / 实现

- **R 语言 `relaimpo` 包**（Ulrike Grömping）：提供 LMG 指标，对所有变量进入次序平均分摊方差，是相对权重/Shapley 同类思路的开源实现。
- **Displayr**：商业分析方案，默认推荐 Johnson 相对权重或 Shapley value，可直接输出加总为 100% 的稳健重要性。
- **Shapley value 回归**：源自博弈论，对所有变量进入顺序求平均贡献，结果跨样本更稳，是处理相关自变量的稳健方案。

## 局限与误用

- **把相关当因果**：高重要性只说明该属性与总体满意度同向变化，可能存在共同前因或反向因果，不能直接断言"改了它满意度必涨"。
- **多重共线性致系数失真**：属性高度相关时用原始回归 beta 排序，权重会不稳、变号；应改用相对权重 / Shapley。
- **只看重要性、不看表现**：高重要但已做得很好的属性继续砸资源，是典型过度投入；必须叠加 IPA 表现维度。
- **属性清单残缺**：漏掉关键属性会让模型把其作用错配给相邻属性，结论偏差。
- **轻视基本型属性**：基本型在派生重要性上常偏低，但失守会塌方，需用 Kano 兜底。
- **样本与量表问题**：样本量不足、量表区分度差或天花板效应，都会让重要性估计不可靠。
- **用陈述重要性替代派生重要性**：直接问"重要吗"区分度低且失真，应以数据派生为主、陈述为辅做对照。
