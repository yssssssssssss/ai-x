用研 AI 专项 · 能力模块清单图

本图仅列出项目具备的能力模块,不表达模块之间的调用顺序或流程关系。实际执行时由 Orchestrator 根据用户需求动态组合。

一、用户入口与任务管理

  • ChatGPT 式用研任务对话工作台
  • 新建任务、历史任务、任务恢复
  • 能力库与案例库查看入口
  • 计划确认、执行进度、最终报告展示
  • 用户反馈与采纳记录

二、需求理解能力

  • 一句话需求结构化为 ResearchTask
  • 识别 task_type、业务领域、研究目标
  • 识别目标用户、数据来源、限制条件
  • 缺失信息分级:可假设、需确认、必须阻断
  • 敏感数据与风险等级识别

三、决策与方案编排能力

  • 决策节点池按任务类型动态激活
  • 不固定 7 步,不做硬编码流程
  • 生成待确认执行计划
  • 支持用户修改假设与补充信息
  • 计划确认后才执行 skill/tool

四、Skill 能力库

  • 竞品分析 Skill
  • 用户之声/VOC 诊断 Skill
  • 人群审查与样本合理性 Skill
  • 体验走查、UI 审查、眼动 review Skill
  • 研究计划、访谈提纲、报告生成 Skill

五、Tool 执行能力

  • o2 查询与自动化操作
  • 竞品页面抓取与页面截图
  • 用户反馈、VOC、业务数据查询
  • 文件处理、报告导出、Wiki 沉淀
  • 内部 API / MCP / script 适配扩展

六、知识库能力

  • 用研方法论与判断标准
  • 历史案例、优秀报告、失败案例
  • 研究计划、访谈提纲、报告模板
  • 业务术语、人群标签、用户分层
  • metadata + BM25/关键词检索,MVP 后可接 pgvector

七、执行与运行记忆能力

  • 按确认计划执行 skill/tool
  • 记录 execution_log、context_manifest、trace_id
  • 运行工作区保存 plan、tool_outputs、failures、artifacts
  • 支持失败复盘、重试、跳过、终止
  • Research Memory DB 保存会话、任务、决策、执行、产物、反馈

八、质量与安全治理能力

  • ResearchTask、Plan、DecisionState、Report schema 校验
  • registry / risk / report / context linter
  • 高风险 tool 审批与 blocked 状态
  • PII / confidential 数据分级、脱敏与审批
  • 来源标注:用户输入、知识库、Tool、LLM 推断、待人工确认

九、能力接入与运营能力

  • Skill/Tool manifest + schema + examples + evals 接入
  • draft / active / deprecated 状态管理
  • owner、reviewer、approver 角色语义
  • 能力迁移 checklist 与种子 skill 模板
  • 后续支持 skill 质量评估、成本看板、文档园丁/规则园丁

核心原则:配置、skill、tool、知识库放 Git;运行时数据放 DB;大文件和中间产物放对象存储;Orchestrator 负责加载、校验、调用、留痕,不把业务判断写死在代码里。
