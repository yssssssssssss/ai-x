### 技术突破

M1-B 突破的是**"o2 CLI 生态接通"**这道墙:

1. **实现 O2LaunchAdapter**([tool-adapter.ts](apps/orchestrator-runtime/src/runtime/tool-adapter.ts)) — 通过 `child_process.spawn('o2', ['launch', cli, sub, ...args, '--output', 'json'])` 调 o2 CLI,读 stdout JSON,按 `o2_output_path` 提取主字段
2. **新增 jd-product-search tool** — manifest 声明 `o2_cli=metasearch, o2_subcommand=search, o2_output_path=data.products`;走真实 metasearch(京东 A2A Gateway sku-search agent)
3. **skill required_tools 更新** — 让 2 个竞品 skill 声明 `jd-product-search`,domain 预筛能保留它并让 LLM 选中
4. **修 cleanStep 兜底** — llm/reviewer 步 LLM 常不给 `actor_id`,用 actor_type 名兜底,避免 schema 校验挂
5. **诚实标注 o2-web-search 是 fake** — 不再假装,把 adapter_type 改成 fake,让新 tool 名副其实占用真实 o2 通道

### 产品形态

现在系统真正具备"**双通道**":

```
用户输入 query
   ├─ 场景需要真实素材(设计稿/竞品资料)
   │  └─ pendingUploads 让用户上传 → tool/skill 消费真素材
```
