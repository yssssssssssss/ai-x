用研 AI 专项 · 核心运行流程图

四段协作流:任务理解 → 待确认计划 → 执行 → 交付。计划确认是硬闸门,未确认不执行 skill/tool。

sequenceDiagram
  participant U as 用户
  participant UI as Web门户/咚咚Bot
  participant API as agent-api
  participant OR as Orchestrator Runtime
  participant CFG as Registry/DecisionGraph/Knowledge
  participant LLM as LLM Client
  participant TOOL as Tool Adapter
  participant DB as Research Memory DB
  participant WS as Run Workspace/Object Store

  U->>UI: 输入一句用研需求
  UI->>API: 提交消息
  API->>DB: 写 conversations / messages
  API->>OR: 创建 research_task

  OR->>LLM: 结构化理解用户需求
  LLM-->>OR: ResearchTask JSON
  OR->>DB: 写 research_tasks

  OR->>CFG: 读取 decision-graph / skill-registry / tool-registry
  OR->>LLM: 激活决策节点 + 选择候选 skill/tool
  LLM-->>OR: DecisionState + 候选能力
  OR->>DB: 写 task_decision_states

  OR->>CFG: 渐进加载候选 SKILL.md
  OR->>LLM: 生成待确认执行计划
  LLM-->>OR: Execution Plan
  OR->>WS: 写 plan.json
  OR->>API: 返回待确认计划
  API->>UI: 展示计划卡片

  U->>UI: 确认计划 / 修改假设
  UI->>API: 确认执行
  API->>OR: 开始执行

  loop 每个执行步骤
    OR->>DB: 写 execution_log = running
    OR->>CFG: 按需加载 schema / examples / templates
    OR->>TOOL: 调用 skill/tool
    TOOL-->>OR: Tool output / error
    OR->>WS: 写 tool_outputs / context_manifest / failures
    OR->>DB: 更新 execution_log status
  end

  OR->>LLM: synthesis 汇总报告
  LLM-->>OR: Research Report
  OR->>DB: 写 artifacts
  OR->>API: 返回最终报告
  API->>UI: 展示报告 + 来源标注 + 反馈入口

  U->>UI: 提交反馈
  UI->>API: rating / adopted / comment
  API->>DB: 写 user_feedback
失败处理:步骤失败只停在该步骤,写 failures.jsonl 与 context_manifest_ref,人工确认后重试、跳过或终止。
