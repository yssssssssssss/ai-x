## 1. 核心架构认知(避免新会话重新探索)

### 1.1 四段流(orchestrator.ts)
`planPhase`(理解→节点激活→知识召回→状态判定→候选生成,产 depth/speed 2 份候选)→ `selectPlan`(选一份,装配 execution-plan)→ `executePhase`(逐步执行)→ `finalizeReport`(合成报告)。

### 1.2 执行引擎(executePhase 已重构为)
- `runFrom(startIdx, ctx)`:顺序 for 循环,**串行**执行(步间有数据依赖)
- `runStep(step, ctx)`:按 actor_type 分派 tool/skill/llm/reviewer
- `finalizeReport(ctx)`:数据闸门(toolOutputs 空则抛 AllStepsFailedError)+ synthesis
- **容错**:失败不整链崩,停在该步 `paused` + 落 run_state.json,`resumePhase(skip/abort)` 断点续跑
- 终态:completed / completed_with_gaps / failed / paused

### 1.3 能力分两类(重要认知)
- **采集类**(去外部拿数据):tavily-web-search / jd-product-search / ai-spider-search / o2-web-search(fake)
- **分析类**(处理用户输入,不采集):5 个 lab(aesthetic=纯像素;attention/vision-brand=像素+VLM;experience=本地模型库;virtual-user=LLM模拟)

## 4. 踩过的坑(别再踩)

1. **真实 LLM 给 step 字段极不规范**:用 step_id 非 step_no、漏 step_name、input 给字符串/空对象、llm/reviewer 步不给 actor_id、assumptions 给字符串数组 → **全靠 cleanStep 兜底**,改 cleanStep 时别删这些防御
2. **fetch 5 分钟 headers timeout**:12 步执行 >5min,前台 fetch 会断 → 测试脚本改**轮询磁盘 report.json**,不 await execute
3. **tsx 顶层 await**:脚本必须放 `scripts/`(有 tsconfig),放 `/tmp` 会报 "Top-level await not supported with cjs"
4. **text base64 直喂 LLM 会超时**:22KB SKILL.md + base64 资料 → LLM 解码耗时触发 gateway timeout。已在 runStep 解码 text/* 为明文
5. **.env 改动 agent-api 不自动生效**:tsx watch 只热载代码,改 .env 要 `pkill -f "tsx watch apps/agent-api"; npm run api:dev`
6. **agent-api 连的是真实 gateway**(内网可达),plan/execute 是真跑,慢(plan~50s,execute 12步~6min)
7. **grep 在这个终端有渲染异常**,用 awk/python 替代
8. **skill-registry.yaml 447 行超读取预算**,用 awk/sed/python 精确定位,别整读
