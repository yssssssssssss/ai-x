# 用研 AI M1 · 业务专家试用招募包(2026-07-20)

> **给发起人**:本文是**素材草稿**,含试用手册 + 邀请函 + 反馈问卷三部分,可整体转发,也可拆开分别发到咚咚/邮件/JoySpace。发前请:①补联系人 ②补试用截止日期 ③按招募对象口味微调邀请函措辞。
> **核心目标**:回答一个问题——「这系统现在能不能真帮到用研/设计/竞品同事日常工作」。技术能跑通已被内部验证,现在缺的是**业务视角的可采纳性判断**。

## 第一部分 · 试用手册(给受邀者看)

### 一句话背景

**用研 AI = query 驱动的 Agent 平台**,聚合各团队用户研究能力(竞品分析 / 用户访谈提纲 / 满意度诊断 / 旅程图 / 无障碍走查 等 22 项 skill),你用一句话说明诉求,系统自动排研究流水线并产出结构化报告。

### 访问入口

- **前端**:`http://localhost:5173`(或团队部署环境的对应地址,发前请补)
- **测试账号**:`tester+ui@local.dev` / 密码 `testtest`
- **服务窗口**:发起人告知的时间范围内(agent-api 在跑)

### 5 分钟上手

1. 登录后进「工作台」,顶部输入框写你**真实工作里的一个诉求**(越具体越好),点执行
2. 系统会给你 2 份候选方案:**深度优先**(方法论完整,~10 分钟)vs **速度优先**(2-3 分钟出结论)
3. 挑一个,若需要上传设计稿/资料按提示补充,点确认
4. 等待 3-15 分钟(取决于复杂度),报告出来会自动展示
5. **每条 finding 都带溯源**,可点开看数据来自哪个 tool/skill

### 系统当前能做的 6 类 query 举例

| 场景 | 举例 query |
|---|---|
| **竞品研究** | "对硅基智能、小冰、Live-Persona 三家数字人做竞品分析,查各家能力/商业模式/近半年动态" |
| **访谈提纲** | "给直播运营做用户访谈提纲,3 个研究问题:选品痛点、竞品体验、付费意愿" |
| **旅程图** | "画一份『用户在京东买生鲜从看到收货』的用户旅程图,含情绪曲线、机会点" |
| **满意度诊断** | "本季度 NSS 掉了 4 分,给一份归因诊断和改进优先级" |
| **设计走查** | 上传设计稿 → "对这版首页做美学 / 注意力 / 品牌视觉三维走查" |
| **VOC 编码** | 上传一批评价文本 → "帮我编码归类,找出 top 痛点" |

### 我们已知的局限(诚实说)

- 22 skill 里**只有 5 个做过完整端到端验证**(digital-human 竞品 / competitive-web-research / competitive-app-analysis / design-experience-review / generate-interview-guide),其余 17 个可能会有**结构不规整、方法引用漂**等偶发问题
- **京东内部数据能力(采销工作台/黄金眼)当前只有部分权限**——涉及 comment/sku-selection/ge-search 的 query 可能拿不到真实数据
- 复杂 query(涉及多轮、多 skill 组合)执行时间可能到 **10-20 分钟**,请耐心;若 UI 长时间无进度可刷新
- 报告里若出现 `llm_inference` 标记的结论,说明是模型基于经验的推断,**不是从工具数据来的事实**——请重点审这些
- 产出的报告我们**不会自动发给任何人**,只有你自己看得到,任何数据都在本地

### 遇到问题怎么办

- **数据不对 / 结论离谱**:直接在反馈问卷里指出,越具体越好
- **系统卡死 / 报错**:截图 + 你输入的 query 发给发起人(**请勿分享账号密码**)
- **想跑但没时间**:也可以只跑 1 个 case + 填 A 部分反馈,不必勉强

嗨 [名字],

我们在做一个内部工具叫「用研 AI」——给你的日常研究工作做加速。它现在已经能:

- 你说一句话,它自动规划研究流水线(方法组合 / 用哪些 skill / 采集哪些数据)
- 端到端跑通,产出**带溯源的结构化报告**(每条结论能追到源数据)
- 覆盖 22 个用研 skill(竞品 / 访谈提纲 / 旅程图 / VOC / 满意度诊断 / 设计走查 / 无障碍…)

技术层能跑通,我们已经反复验证过。**但我们现在最需要知道的是——你,作为真实用研工作者,觉得它能不能帮你干活**。

**邀请你花 30-45 分钟做一次试用**:
- 15 分钟:用你**日常真实工作**里的 1-2 个诉求,给它跑一遍(不用编造场景,越真越好)

我们承诺:
- **不追责** —— 说难听话我们才有收益;说好话我们反而学不到东西
- **不点名** —— 反馈整体使用,不会指名道姓地在评审会上说"某某觉得…"
- **结果仅用于内部决策** —— 决定这个项目要不要继续投、往哪个方向走

时间投入:总共 30-45 分钟。可以你方便的时候单独跑,不需要拉会议。

**试用手册在附件**(或此后附)。有问题随时找我。

## 附录 · 试用效果的量化门槛(内部参考,不给受邀者看)

来自项目全景文档中的「业务盲评门禁」定义:

| 指标 | 达标线 | 计算方式 |
|---|---|---|
| **可采纳率** | ≥ 70% | 「可采纳」计数 / 有效反馈总数 |
| **溯源率评分均值** | ≥ 4.0 / 5 | Q5 均值 |
| **业务价值感受(小时/周)** | 中位数 ≥ 3 | 隐藏题中位数 |
| **推荐率** | ≥ 50% | Q9「会推荐」占比 |

**任意两项达标即可判定"值得继续投入";全达标可以进 M2 阶段。**

## 使用注意

- 本招募包**不要一次给太多人**——建议先发 3-5 个内部同事(不同岗位),收到反馈修 1 轮再扩到 8-15 个
- 试用**在同一时段**做效率最高(用研同事一起跑,凑数据快)——但不强制
- 反馈问卷可以做成 JoySpace 表单 / 腾讯问卷,收上来自动结构化(比自由 markdown 更好统计)
- 我们**不承诺时间点交付** —— 收到反馈后按重要性优先级修,而不是硬性 sprint
